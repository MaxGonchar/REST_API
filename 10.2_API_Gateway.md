# 10.2 API Gateway

## The Problem

You've built a system with multiple microservices. But now you face challenges:

**Without API Gateway - Direct client to microservices:**

```
Mobile App ──┬──> User Service (auth)
             ├──> Product Service (catalog)
             ├──> Order Service (checkout)
             ├──> Payment Service (billing)
             └──> Inventory Service (stock)

Problems:
❌ Clients need to know all service URLs
❌ Authentication logic duplicated in every service
❌ Each service handles its own rate limiting
❌ CORS configuration in every service
❌ Multiple network calls from client (slow on mobile)
❌ Services can't change URLs without breaking clients
❌ No unified logging/monitoring
❌ Client needs to handle different protocols (REST, gRPC)
```

**Real scenario that goes wrong:**

```javascript
// Mobile app needs to show user dashboard
// Requires data from 5 different services

async function loadDashboard() {
  // Problem: 5 separate HTTP requests
  const user = await fetch('https://user-service.com/api/users/me');
  const orders = await fetch('https://order-service.com/api/orders?userId=' + user.id);
  const recommendations = await fetch('https://rec-service.com/api/recommendations/' + user.id);
  const cart = await fetch('https://cart-service.com/api/carts/' + user.id);
  const notifications = await fetch('https://notif-service.com/api/notifications/' + user.id);
  
  // Problems:
  // ❌ 5 network roundtrips (slow, especially on mobile/3G)
  // ❌ Each service authenticates separately (wasteful)
  // ❌ If user-service.com changes domain, app breaks
  // ❌ No rate limiting across services (can overwhelm backend)
  // ❌ Different error formats from each service
  // ❌ Client needs to handle retries for each service
}
```

**Solution: API Gateway**

An **API Gateway** is a single entry point that sits between clients and backend services, handling cross-cutting concerns.

```
Mobile App ──────> API Gateway ──┬──> User Service
                                 ├──> Product Service
                                 ├──> Order Service
                                 ├──> Payment Service
                                 └──> Inventory Service

Benefits:
✅ Single URL for clients (gateway.example.com)
✅ Authentication in one place
✅ Rate limiting across all services
✅ Request aggregation (combine multiple service calls)
✅ Protocol translation (REST → gRPC)
✅ Unified logging and monitoring
✅ Service discovery (services can move)
✅ Client-friendly responses
```

---

## What is an API Gateway?

An **API Gateway** is a server that acts as a reverse proxy, routing client requests to the appropriate backend services while providing additional functionality.

### Core Capabilities

**1. Routing**
- Route requests to correct backend service
- Path-based: `/users/*` → User Service, `/products/*` → Product Service
- Support for versioning: `/v1/*` → Old services, `/v2/*` → New services

**2. Authentication & Authorization**
- Validate tokens once at gateway
- Forward user context to services
- Prevent unauthenticated requests from reaching services

**3. Rate Limiting**
- Throttle requests per client
- Protect backend from overload
- Different limits for different tiers (free vs premium)

**4. Request/Response Transformation**
- Modify headers, body, query parameters
- Aggregate responses from multiple services
- Protocol translation (REST ↔ gRPC)

**5. Caching**
- Cache frequent responses
- Reduce load on backend services
- Improve response time

**6. Load Balancing**
- Distribute traffic across service instances
- Health checks
- Failover

**7. Observability**
- Centralized logging
- Metrics and analytics
- Distributed tracing

## How API Gateway Works

### Architecture

```
┌──────────────────────────────────────────────────────────────┐
│                       API Gateway                            │
│                                                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   Auth      │  │Rate Limiter │  │   Cache     │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │              Router                                 │     │
│  │  /api/users/*    → User Service                    │     │
│  │  /api/products/* → Product Service                 │     │
│  │  /api/orders/*   → Order Service                   │     │
│  └────────────────────────────────────────────────────┘     │
│                                                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   Logging   │  │   Metrics   │  │ Transform   │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└──────────────────────────────────────────────────────────────┘
         │                  │                  │
         ↓                  ↓                  ↓
   User Service      Product Service      Order Service
```

### Request Flow

```
1. Client → API Gateway
   GET /api/users/123
   Authorization: Bearer abc123

2. Gateway: Authenticate
   ✓ Validate JWT token
   ✓ Extract user ID
   
3. Gateway: Rate Limit Check
   ✓ Check request count for this user
   ✓ Allow or reject
   
4. Gateway: Route
   ✓ Match /api/users/* → User Service
   ✓ Forward to http://user-service:3000/users/123
   
5. Service: Process
   ✓ User Service handles request
   ✓ Returns user data
   
6. Gateway: Transform Response
   ✓ Add headers
   ✓ Log request
   ✓ Cache result
   
7. Client ← API Gateway
   200 OK
   {user data}
```

## Building a Simple API Gateway (Node.js)

### Basic Gateway with Routing

```javascript
const express = require('express');
const axios = require('axios');
const app = express();

app.use(express.json());

// Service registry - map routes to backend services
const services = {
  users: 'http://localhost:3001',
  products: 'http://localhost:3002',
  orders: 'http://localhost:3003'
};

// Basic routing middleware
app.use('/api/users/*', async (req, res) => {
  await proxyRequest('users', req, res);
});

app.use('/api/products/*', async (req, res) => {
  await proxyRequest('products', req, res);
});

app.use('/api/orders/*', async (req, res) => {
  await proxyRequest('orders', req, res);
});

// Proxy function - forwards request to backend service
async function proxyRequest(serviceName, req, res) {
  const serviceUrl = services[serviceName];
  
  // Remove /api/{service} prefix for backend
  const path = req.originalUrl.replace(`/api/${serviceName}`, '');
  const targetUrl = `${serviceUrl}${path}`;
  
  try {
    console.log(`Proxying ${req.method} ${req.originalUrl} → ${targetUrl}`);
    
    // Forward request to backend service
    const response = await axios({
      method: req.method,
      url: targetUrl,
      headers: req.headers,
      data: req.body,
      params: req.query
    });
    
    // Return backend response to client
    res.status(response.status).json(response.data);
    
  } catch (error) {
    if (error.response) {
      // Backend returned an error
      res.status(error.response.status).json(error.response.data);
    } else {
      // Network error or service unavailable
      res.status(503).json({
        error: 'Service unavailable',
        service: serviceName
      });
    }
  }
}

app.listen(8080, () => {
  console.log('API Gateway running on port 8080');
});
```

**How it works:**
1. Client calls: `GET /api/users/123`
2. Gateway matches `/api/users/*` route
3. Proxies to: `http://localhost:3001/123`
4. Returns service response to client

**Benefits:**
- Clients only know `http://gateway:8080`
- Backend services can change URLs
- Central point for logging

## Adding Authentication

```javascript
const jwt = require('jsonwebtoken');

const SECRET_KEY = 'your-secret-key';

// Authentication middleware
async function authenticate(req, res, next) {
  // Extract token from Authorization header
  const authHeader = req.headers.authorization;
  
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return res.status(401).json({
      error: 'Authentication required',
      message: 'Missing or invalid Authorization header'
    });
  }
  
  const token = authHeader.substring(7); // Remove 'Bearer '
  
  try {
    // Verify JWT token
    const decoded = jwt.verify(token, SECRET_KEY);
    
    // Attach user info to request
    req.user = {
      id: decoded.userId,
      email: decoded.email,
      role: decoded.role
    };
    
    // Add user context to headers for backend services
    req.headers['x-user-id'] = decoded.userId;
    req.headers['x-user-role'] = decoded.role;
    
    console.log(`Authenticated user: ${decoded.email}`);
    next();
    
  } catch (error) {
    if (error.name === 'TokenExpiredError') {
      return res.status(401).json({
        error: 'Token expired',
        message: 'Please login again'
      });
    }
    
    return res.status(401).json({
      error: 'Invalid token',
      message: error.message
    });
  }
}

// Public routes (no authentication needed)
app.post('/api/auth/login', async (req, res) => {
  // Proxy to auth service
  await proxyRequest('auth', req, res);
});

app.post('/api/auth/register', async (req, res) => {
  await proxyRequest('auth', req, res);
});

// Protected routes (authentication required)
app.use('/api/users/*', authenticate, async (req, res) => {
  await proxyRequest('users', req, res);
});

app.use('/api/orders/*', authenticate, async (req, res) => {
  await proxyRequest('orders', req, res);
});

// Products can be viewed without auth
app.use('/api/products/*', async (req, res) => {
  await proxyRequest('products', req, res);
});
```

**Benefits:**
- Authentication logic in one place (not duplicated in every service)
- Backend services receive authenticated user info in headers
- Services can trust `x-user-id` header (gateway verified it)
- Easy to change auth strategy (OAuth, JWT, API keys) without touching services

## Adding Rate Limiting

```javascript
// Simple in-memory rate limiter
class RateLimiter {
  constructor() {
    this.requests = new Map(); // userId -> [timestamps]
  }
  
  /**
   * Check if user exceeded rate limit
   * @param {string} userId - User ID
   * @param {number} maxRequests - Max requests allowed
   * @param {number} windowMs - Time window in milliseconds
   * @returns {boolean} - true if allowed, false if rate limited
   */
  isAllowed(userId, maxRequests, windowMs) {
    const now = Date.now();
    const userRequests = this.requests.get(userId) || [];
    
    // Remove requests outside time window
    const recentRequests = userRequests.filter(timestamp => 
      now - timestamp < windowMs
    );
    
    if (recentRequests.length >= maxRequests) {
      return false; // Rate limit exceeded
    }
    
    // Add current request
    recentRequests.push(now);
    this.requests.set(userId, recentRequests);
    
    return true; // Allowed
  }
  
  /**
   * Get remaining requests for user
   */
  getRemaining(userId, maxRequests, windowMs) {
    const now = Date.now();
    const userRequests = this.requests.get(userId) || [];
    
    const recentRequests = userRequests.filter(timestamp => 
      now - timestamp < windowMs
    );
    
    return Math.max(0, maxRequests - recentRequests.length);
  }
}

const rateLimiter = new RateLimiter();

// Rate limiting middleware
function rateLimit(maxRequests = 100, windowMs = 60000) {
  return (req, res, next) => {
    // Use user ID if authenticated, else IP address
    const identifier = req.user?.id || req.ip;
    
    if (!rateLimiter.isAllowed(identifier, maxRequests, windowMs)) {
      const remaining = rateLimiter.getRemaining(identifier, maxRequests, windowMs);
      const resetTime = Math.ceil(windowMs / 1000);
      
      res.set({
        'X-RateLimit-Limit': maxRequests,
        'X-RateLimit-Remaining': remaining,
        'X-RateLimit-Reset': resetTime
      });
      
      return res.status(429).json({
        error: 'Too many requests',
        message: `Rate limit exceeded. Try again in ${resetTime} seconds.`,
        limit: maxRequests,
        window: `${windowMs / 1000} seconds`
      });
    }
    
    // Add rate limit headers to response
    const remaining = rateLimiter.getRemaining(identifier, maxRequests, windowMs);
    res.set({
      'X-RateLimit-Limit': maxRequests,
      'X-RateLimit-Remaining': remaining
    });
    
    next();
  };
}

// Apply rate limiting to all routes
app.use(rateLimit(100, 60000)); // 100 requests per minute

// Different rate limits for different tiers
app.use('/api/premium/*', authenticate, rateLimit(1000, 60000)); // Premium: 1000/min
app.use('/api/free/*', authenticate, rateLimit(60, 60000));      // Free: 60/min
```

**Benefits:**
- Protects backend from overload
- Fair usage across users
- Easy to implement tiered plans (free vs premium)
- Responds before hitting backend (saves resources)

## Request Aggregation (Combining Multiple Services)

```javascript
// Aggregate endpoint - combines data from multiple services
app.get('/api/dashboard', authenticate, async (req, res) => {
  const userId = req.user.id;
  
  try {
    console.log(`Loading dashboard for user ${userId}`);
    
    // Call multiple services in parallel
    const [userResponse, ordersResponse, recommendationsResponse] = await Promise.all([
      axios.get(`${services.users}/users/${userId}`, {
        headers: { 'x-user-id': userId }
      }),
      axios.get(`${services.orders}/orders?userId=${userId}&limit=5`, {
        headers: { 'x-user-id': userId }
      }),
      axios.get(`${services.products}/recommendations/${userId}`, {
        headers: { 'x-user-id': userId }
      })
    ]);
    
    // Combine responses into single response
    res.json({
      user: userResponse.data,
      recentOrders: ordersResponse.data,
      recommendations: recommendationsResponse.data,
      _meta: {
        timestamp: new Date().toISOString(),
        sources: ['users', 'orders', 'products']
      }
    });
    
  } catch (error) {
    console.error('Dashboard aggregation error:', error.message);
    
    res.status(500).json({
      error: 'Failed to load dashboard',
      message: 'One or more services unavailable'
    });
  }
});
```

**Before (without gateway):**
```javascript
// Client makes 3 separate requests
const user = await fetch('/api/users/123');
const orders = await fetch('/api/orders?userId=123');
const recommendations = await fetch('/api/recommendations/123');

// Problems:
// - 3 HTTP roundtrips (slow, especially on mobile)
// - 3 separate authentication checks
// - Client handles combining data
```

**After (with gateway aggregation):**
```javascript
// Client makes 1 request
const dashboard = await fetch('/api/dashboard');

// Benefits:
// ✅ 1 HTTP roundtrip (faster)
// ✅ 1 authentication check
// ✅ Gateway handles complexity
// ✅ Better mobile experience
```

## Request/Response Transformation

```javascript
// Transform request before sending to backend
app.post('/api/v2/orders', authenticate, async (req, res) => {
  // Client sends v2 format
  const clientRequest = req.body;
  // {
  //   items: [{productId: "p1", quantity: 2}],
  //   delivery: {address: "...", method: "express"}
  // }
  
  // Backend expects v1 format - transform it
  const backendRequest = {
    user_id: req.user.id,
    products: clientRequest.items.map(item => ({
      product_id: item.productId,
      qty: item.quantity
    })),
    shipping_address: clientRequest.delivery.address,
    shipping_method: clientRequest.delivery.method,
    created_at: new Date().toISOString()
  };
  
  try {
    const response = await axios.post(
      `${services.orders}/orders`,
      backendRequest,
      { headers: { 'x-user-id': req.user.id } }
    );
    
    // Transform backend response to client format
    const backendResponse = response.data;
    
    res.status(201).json({
      orderId: backendResponse.order_id,
      status: backendResponse.status,
      total: backendResponse.total_amount,
      estimatedDelivery: backendResponse.estimated_delivery_date
    });
    
  } catch (error) {
    res.status(500).json({ error: 'Order creation failed' });
  }
});
```

**Why transformation matters:**
- Client and backend can use different formats
- Support multiple API versions (v1, v2) without changing backend
- Maintain backward compatibility
- Hide internal implementation details

## Adding Caching

```javascript
const NodeCache = require('node-cache');
const cache = new NodeCache({ stdTTL: 300 }); // 5 minute default TTL

// Cache middleware
function cacheMiddleware(ttl = 300) {
  return (req, res, next) => {
    // Only cache GET requests
    if (req.method !== 'GET') {
      return next();
    }
    
    // Create cache key from URL and user ID
    const cacheKey = `${req.user?.id || 'anon'}:${req.originalUrl}`;
    
    // Check if cached
    const cachedResponse = cache.get(cacheKey);
    
    if (cachedResponse) {
      console.log(`Cache HIT: ${cacheKey}`);
      res.set('X-Cache', 'HIT');
      return res.json(cachedResponse);
    }
    
    console.log(`Cache MISS: ${cacheKey}`);
    
    // Override res.json to cache response
    const originalJson = res.json.bind(res);
    res.json = (body) => {
      // Cache the response
      cache.set(cacheKey, body, ttl);
      res.set('X-Cache', 'MISS');
      return originalJson(body);
    };
    
    next();
  };
}

// Apply caching to product routes (products change infrequently)
app.use('/api/products', cacheMiddleware(600)); // Cache for 10 minutes

// Don't cache user-specific data
app.use('/api/users/*', authenticate, async (req, res) => {
  await proxyRequest('users', req, res);
});

// Invalidate cache when data changes
app.post('/api/products', authenticate, async (req, res) => {
  // Create product
  await proxyRequest('products', req, res);
  
  // Invalidate product list cache
  cache.flushAll();
});
```

**Benefits:**
- Reduces load on backend services
- Faster response times
- Better user experience
- Saves database queries

## Complete API Gateway Implementation

Here's a production-ready gateway with all features:

```javascript
const express = require('express');
const axios = require('axios');
const jwt = require('jsonwebtoken');
const NodeCache = require('node-cache');

const app = express();
app.use(express.json());

// Configuration
const config = {
  jwtSecret: process.env.JWT_SECRET || 'your-secret-key',
  services: {
    auth: process.env.AUTH_SERVICE || 'http://localhost:3001',
    users: process.env.USER_SERVICE || 'http://localhost:3002',
    products: process.env.PRODUCT_SERVICE || 'http://localhost:3003',
    orders: process.env.ORDER_SERVICE || 'http://localhost:3004'
  },
  rateLimit: {
    free: { max: 60, window: 60000 },      // 60 requests/minute
    premium: { max: 1000, window: 60000 }  // 1000 requests/minute
  },
  cache: {
    ttl: 300 // 5 minutes
  }
};

// Initialize cache and rate limiter
const cache = new NodeCache({ stdTTL: config.cache.ttl });
const rateLimiter = new RateLimiter();

// ==================== Middleware ====================

// Request logging
app.use((req, res, next) => {
  const start = Date.now();
  
  res.on('finish', () => {
    const duration = Date.now() - start;
    console.log(
      `[${new Date().toISOString()}] ` +
      `${req.method} ${req.originalUrl} ` +
      `${res.statusCode} ${duration}ms ` +
      `user=${req.user?.id || 'anon'}`
    );
  });
  
  next();
});

// CORS
app.use((req, res, next) => {
  res.header('Access-Control-Allow-Origin', '*');
  res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, PATCH, DELETE, OPTIONS');
  res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization');
  
  if (req.method === 'OPTIONS') {
    return res.sendStatus(200);
  }
  
  next();
});

// Authentication middleware
function authenticate(req, res, next) {
  const authHeader = req.headers.authorization;
  
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return res.status(401).json({
      error: 'Authentication required'
    });
  }
  
  const token = authHeader.substring(7);
  
  try {
    const decoded = jwt.verify(token, config.jwtSecret);
    req.user = {
      id: decoded.userId,
      email: decoded.email,
      role: decoded.role,
      tier: decoded.tier || 'free'
    };
    
    // Add user context for backend services
    req.headers['x-user-id'] = req.user.id;
    req.headers['x-user-role'] = req.user.role;
    
    next();
  } catch (error) {
    return res.status(401).json({
      error: 'Invalid or expired token'
    });
  }
}

// Optional authentication (for routes that work with or without auth)
function optionalAuth(req, res, next) {
  const authHeader = req.headers.authorization;
  
  if (authHeader && authHeader.startsWith('Bearer ')) {
    return authenticate(req, res, next);
  }
  
  next();
}

// Rate limiting middleware
function rateLimit(tier = 'free') {
  return (req, res, next) => {
    const limits = config.rateLimit[tier];
    const identifier = req.user?.id || req.ip;
    
    if (!rateLimiter.isAllowed(identifier, limits.max, limits.window)) {
      return res.status(429).json({
        error: 'Rate limit exceeded',
        limit: limits.max,
        window: `${limits.window / 1000}s`
      });
    }
    
    const remaining = rateLimiter.getRemaining(identifier, limits.max, limits.window);
    res.set('X-RateLimit-Remaining', remaining);
    next();
  };
}

// Dynamic rate limiting based on user tier
function dynamicRateLimit(req, res, next) {
  const tier = req.user?.tier || 'free';
  return rateLimit(tier)(req, res, next);
}

// Caching middleware
function cacheResponse(ttl) {
  return (req, res, next) => {
    if (req.method !== 'GET') {
      return next();
    }
    
    const cacheKey = `${req.user?.id || 'anon'}:${req.originalUrl}`;
    const cached = cache.get(cacheKey);
    
    if (cached) {
      res.set('X-Cache', 'HIT');
      return res.json(cached);
    }
    
    const originalJson = res.json.bind(res);
    res.json = (body) => {
      cache.set(cacheKey, body, ttl);
      res.set('X-Cache', 'MISS');
      return originalJson(body);
    };
    
    next();
  };
}

// ==================== Proxy Function ====================

async function proxyRequest(serviceName, req, res) {
  const serviceUrl = config.services[serviceName];
  
  if (!serviceUrl) {
    return res.status(500).json({
      error: 'Service not configured',
      service: serviceName
    });
  }
  
  // Build target URL
  const prefix = `/api/${serviceName}`;
  const path = req.originalUrl.replace(prefix, '');
  const targetUrl = `${serviceUrl}${path}`;
  
  try {
    const response = await axios({
      method: req.method,
      url: targetUrl,
      headers: {
        ...req.headers,
        host: new URL(serviceUrl).host // Override host header
      },
      data: req.body,
      params: req.query,
      timeout: 10000, // 10 second timeout
      validateStatus: () => true // Don't throw on any status code
    });
    
    // Forward response headers
    Object.keys(response.headers).forEach(key => {
      res.set(key, response.headers[key]);
    });
    
    res.status(response.status).json(response.data);
    
  } catch (error) {
    console.error(`Proxy error for ${serviceName}:`, error.message);
    
    if (error.code === 'ECONNREFUSED') {
      return res.status(503).json({
        error: 'Service unavailable',
        service: serviceName,
        message: 'Backend service is not responding'
      });
    }
    
    if (error.code === 'ETIMEDOUT') {
      return res.status(504).json({
        error: 'Gateway timeout',
        service: serviceName,
        message: 'Backend service took too long to respond'
      });
    }
    
    res.status(500).json({
      error: 'Internal gateway error',
      message: error.message
    });
  }
}

// ==================== Routes ====================

// Health check
app.get('/health', (req, res) => {
  res.json({
    status: 'healthy',
    timestamp: new Date().toISOString(),
    uptime: process.uptime()
  });
});

// Public routes (no authentication)
app.use('/api/auth/*', rateLimit('free'), (req, res) => {
  proxyRequest('auth', req, res);
});

// Product browsing (optional auth, cached, rate limited by tier)
app.use('/api/products',
  optionalAuth,
  dynamicRateLimit,
  cacheResponse(600), // Cache for 10 minutes
  (req, res) => {
    proxyRequest('products', req, res);
  }
);

// User routes (authentication required)
app.use('/api/users/*',
  authenticate,
  dynamicRateLimit,
  (req, res) => {
    proxyRequest('users', req, res);
  }
);

// Order routes (authentication required, no caching)
app.use('/api/orders/*',
  authenticate,
  dynamicRateLimit,
  (req, res) => {
    proxyRequest('orders', req, res);
  }
);

// Aggregated dashboard endpoint
app.get('/api/dashboard',
  authenticate,
  dynamicRateLimit,
  async (req, res) => {
    const userId = req.user.id;
    
    try {
      const [user, orders, recommendations] = await Promise.all([
        axios.get(`${config.services.users}/users/${userId}`, {
          headers: { 'x-user-id': userId }
        }),
        axios.get(`${config.services.orders}/orders?userId=${userId}&limit=5`, {
          headers: { 'x-user-id': userId }
        }),
        axios.get(`${config.services.products}/recommendations/${userId}`, {
          headers: { 'x-user-id': userId }
        })
      ]);
      
      res.json({
        user: user.data,
        recentOrders: orders.data,
        recommendations: recommendations.data
      });
    } catch (error) {
      res.status(500).json({
        error: 'Failed to load dashboard'
      });
    }
  }
);

// 404 handler
app.use((req, res) => {
  res.status(404).json({
    error: 'Not found',
    path: req.originalUrl
  });
});

// Error handler
app.use((err, req, res, next) => {
  console.error('Gateway error:', err);
  res.status(500).json({
    error: 'Internal gateway error',
    message: err.message
  });
});

// Start server
const PORT = process.env.PORT || 8080;
app.listen(PORT, () => {
  console.log(`API Gateway running on port ${PORT}`);
  console.log('Services:', config.services);
});

// ==================== Rate Limiter Class ====================

class RateLimiter {
  constructor() {
    this.requests = new Map();
  }
  
  isAllowed(userId, maxRequests, windowMs) {
    const now = Date.now();
    const userRequests = this.requests.get(userId) || [];
    const recentRequests = userRequests.filter(t => now - t < windowMs);
    
    if (recentRequests.length >= maxRequests) {
      return false;
    }
    
    recentRequests.push(now);
    this.requests.set(userId, recentRequests);
    return true;
  }
  
  getRemaining(userId, maxRequests, windowMs) {
    const now = Date.now();
    const userRequests = this.requests.get(userId) || [];
    const recentRequests = userRequests.filter(t => now - t < windowMs);
    return Math.max(0, maxRequests - recentRequests.length);
  }
}
```

**Kong** is a popular open-source API gateway built on NGINX and Lua.


**Pros:**
- ✅ High performance (NGINX-based)
- ✅ Rich plugin ecosystem
- ✅ Good documentation
- ✅ Cloud-native (Kubernetes-ready)

**Cons:**
- ❌ Complex configuration
- ❌ Lua for custom plugins (learning curve)

### 2. AWS API Gateway (Cloud Service)

**Managed service** from AWS.

```javascript
// AWS API Gateway with Lambda backend

  "paths": {
    "/users": {

        "x-amazon-apigateway-integration": {
          "uri": "arn:aws:lambda:us-east-1:123456789:function:getUsers",
```

**Pros:**

- ✅ Integrates with AWS services

**Cons:**
- ❌ AWS vendor lock-in
- ❌ Can be expensive at scale
- ❌ Learning curve for AWS ecosystem

### 3. NGINX (Reverse Proxy)

**NGINX** can be configured as an API gateway.

```nginx
# nginx.conf
upstream user_service {
    server user-service:3000;
}

upstream product_service {
    server product-service:3000;
}

server {
    listen 80;
    
    location /api/users/ {
        proxy_pass http://user_service/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /api/products/ {
        proxy_pass http://product_service/;
    }
    
    # Rate limiting
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    
    location /api/ {
        limit_req zone=api_limit burst=20;
    }
}
```

**Pros:**
- ✅ Battle-tested
- ✅ High performance
- ✅ Flexible configuration

**Cons:**
- ❌ Configuration can be complex
- ❌ Limited built-in API gateway features
- ❌ Requires additional tools for auth, etc.

### 4. Express Gateway (Node.js)

**Express Gateway** is built on Express.js.


**Cons:**

## Advanced Patterns

### 1. Backend for Frontend (BFF)

Different clients (web, mobile, IoT) have different needs. Create separate gateway endpoints for each.

```
┌─────────┐     ┌──────────────┐
│ Web App │────>│ Web Gateway  │─┬─> User Service
└─────────┘     └──────────────┘ │
                                 ├─> Product Service
┌──────────┐    ┌───────────────┐│
│Mobile App│───>│Mobile Gateway ││
└──────────┘    └───────────────┘│
                                 │
┌────────┐      ┌──────────────┐ │
│IoT Dev │─────>│ IoT Gateway  │─┘
└────────┘      └──────────────┘
```

```javascript
// Web BFF - returns full data
app.get('/web/api/dashboard', async (req, res) => {
  const [user, orders, products, analytics] = await Promise.all([
    getUser(), getOrders(), getProducts(), getAnalytics()
  ]);
  res.json({ user, orders, products, analytics }); // Full data
});

// Mobile BFF - returns minimal data (save bandwidth)
app.get('/mobile/api/dashboard', async (req, res) => {
  const [user, recentOrders] = await Promise.all([
    getUser(), getRecentOrders(3)
  ]);
  res.json({
    user: { id: user.id, name: user.name }, // Only needed fields
    recentOrders: recentOrders.map(o => ({
      id: o.id,
      status: o.status
    }))
  });
});

// IoT BFF - returns only status codes
app.get('/iot/api/status', async (req, res) => {
  const status = await getSystemStatus();
  res.send(status.code); // Just "OK" or "ERROR"
});
```

### 2. Service Mesh vs API Gateway

**API Gateway**: External traffic (clients → services)
**Service Mesh**: Internal traffic (service → service)

```
External Clients
       ↓
   API Gateway (authentication, rate limiting)
       ↓
   ┌──────────────────────────────┐
   │    Service Mesh (Istio)      │
   │                              │
   │  Service A ↔ Service B       │
   │      ↕           ↕           │
   │  Service C ↔ Service D       │
   │                              │
   │  (mTLS, circuit breaking,    │
   │   retry, timeout)            │
   └──────────────────────────────┘
```

**When you need both:**
- API Gateway: Client-facing (public internet)
- Service Mesh: Service-to-service (within cluster)

## Security Best Practices

```javascript
// 1. Validate JWT strictly
function authenticate(req, res, next) {
  try {
    const token = extractToken(req);
    const decoded = jwt.verify(token, SECRET_KEY, {
      algorithms: ['HS256'], // Specify allowed algorithms
      maxAge: '1h'           // Token must be < 1 hour old
    });
    
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ error: 'Invalid token' });
  }
}

// 2. Sanitize headers before forwarding
function sanitizeHeaders(headers) {
  const dangerous = ['x-forwarded-for', 'x-real-ip'];
  const cleaned = { ...headers };
  dangerous.forEach(h => delete cleaned[h]);
  return cleaned;
}

// 3. Implement request validation
const { body, validationResult } = require('express-validator');

app.post('/api/users',
  body('email').isEmail(),
  body('password').isLength({ min: 8 }),
  (req, res, next) => {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    next();
  },
  proxyToUserService
);

// 4. Add security headers
const helmet = require('helmet');
app.use(helmet());

// 5. Implement IP whitelisting for admin routes
function ipWhitelist(allowedIPs) {
  return (req, res, next) => {
    const clientIP = req.ip;
    if (!allowedIPs.includes(clientIP)) {
      return res.status(403).json({ error: 'IP not allowed' });
    }
    next();
  };
}

app.use('/api/admin/*', ipWhitelist(['10.0.0.1', '10.0.0.2']));
```

## When to Use API Gateway

### ✅ Use API Gateway When:

1. **Microservices architecture** - Multiple backend services need unified entry point
2. **Mobile apps** - Need to reduce number of requests (battery, bandwidth)
3. **Multiple clients** - Web, mobile, IoT with different needs
4. **Cross-cutting concerns** - Auth, rate limiting, logging centralized
5. **Service hiding** - Don't expose internal service URLs to clients
6. **Protocol translation** - REST → gRPC, HTTP → WebSocket
7. **Legacy modernization** - Add modern features to old services

### ❌ Avoid API Gateway When:

1. **Simple monolith** - Single backend service, gateway adds unnecessary complexity
2. **Internal tools** - Trusted internal clients don't need gateway layer
3. **High latency sensitive** - Gateway adds network hop (extra latency)
4. **Small team** - Gateway is extra infrastructure to maintain
5. **Tight coupling needed** - Services need to communicate directly

## Common Pitfalls

### 1. Gateway Becomes Bottleneck

```javascript
// ❌ Bad: Gateway does heavy processing
app.get('/api/reports', async (req, res) => {
  // Don't do expensive operations in gateway!
  const data = await fetchHugeDataset();
  const processed = complexCalculation(data); // Heavy CPU work
  res.json(processed);
});

// ✅ Good: Gateway just routes
app.get('/api/reports', async (req, res) => {
  // Let backend service handle heavy work
  await proxyRequest('reports', req, res);
});
```

### 2. Gateway Has Too Much Logic

```javascript
// ❌ Bad: Gateway contains business logic
app.post('/api/orders', async (req, res) => {
  // Validate inventory
  const product = await checkInventory(req.body.productId);
  if (product.stock < req.body.quantity) {
    return res.status(400).json({ error: 'Out of stock' });
  }
  
  // Calculate discount
  const discount = calculateDiscountLogic(req.user, product);
  
  // Business logic belongs in services, not gateway!
});

// ✅ Good: Gateway just authenticates and routes
app.post('/api/orders', authenticate, async (req, res) => {
  await proxyRequest('orders', req, res);
});
```

### 3. Not Handling Service Failures

```javascript
// ❌ Bad: No timeout, no fallback
const response = await axios.get(serviceUrl);

// ✅ Good: Timeout, circuit breaker, fallback
const response = await axios.get(serviceUrl, {
  timeout: 5000 // 5 second timeout
});

// Even better: Circuit breaker
const breaker = new CircuitBreaker(callService, {
  timeout: 5000,
  errorThresholdPercentage: 50,
  resetTimeout: 30000
});
```

## Key Takeaways

1. **API Gateway is a single entry point** for multiple backend services

2. **Core responsibilities**: Routing, authentication, rate limiting, aggregation, transformation

3. **Benefits**: Simplified client, centralized security, reduced requests, service hiding

4. **Trade-offs**: Extra network hop (latency), single point of failure, added complexity

5. **Keep it thin**: Gateway should route and handle cross-cutting concerns, not business logic

6. **Monitor carefully**: Gateway is critical path - instrument everything

7. **Scale horizontally**: Run multiple gateway instances behind load balancer

8. **Choose based on needs**:
   - Simple project: Build your own with Express
   - Production microservices: Kong or AWS API Gateway
   - High performance: NGINX

9. **Security first**: Validate everything, sanitize headers, rate limit aggressively

10. **Don't over-engineer**: Start simple, add features as needed
