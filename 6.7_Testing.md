# API Testing - Ensuring Quality and Reliability

## Introduction

**API testing** ensures your API works correctly, performs well, and handles errors gracefully. Without testing, you're shipping bugs to production.

**Key principle:** If it's not tested, it's broken.

---

## Types of Testing - What They Mean

### Testing Pyramid

```
           /\
          /  \
         / E2E \         ← Few, slow, expensive
        /______\
       /        \
      /  INTEG   \       ← More, medium speed
     /__________\
    /            \
   /  UNIT TESTS  \      ← Many, fast, cheap
  /________________\
```

### 1. Unit Tests

**What they are:** Test individual functions/methods in isolation.

**What they test:**
- ✅ A single function returns correct output
- ✅ Input validation logic works
- ✅ Helper functions behave correctly
- ✅ Business logic is accurate

**What they DON'T test:**
- ❌ Database interactions
- ❌ External API calls
- ❌ Complete request/response flow

**Example:**

```javascript
// Function to test
function validateEmail(email) {
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return regex.test(email);
}

// Unit test
describe('validateEmail', () => {
  test('should return true for valid email', () => {
    expect(validateEmail('test@example.com')).toBe(true);
  });

  test('should return false for invalid email', () => {
    expect(validateEmail('invalid-email')).toBe(false);
  });

  test('should return false for email without domain', () => {
    expect(validateEmail('test@')).toBe(false);
  });
});
```

### 2. Integration Tests

**What they are:** Test multiple components working together.

**What they test:**
- ✅ API endpoint responds correctly
- ✅ Database queries work
- ✅ Authentication middleware functions
- ✅ Multiple layers interact properly

**What they DON'T test:**
- ❌ Full user workflows across multiple endpoints
- ❌ External third-party services (use mocks)

**Example:**

```javascript
// Integration test
describe('GET /users/:id', () => {
  test('should return user from database', async () => {
    // Setup: Insert test user
    const user = await db.users.create({
      name: 'John Doe',
      email: 'john@example.com'
    });

    // Test: Call API
    const response = await request(app)
      .get(`/users/${user.id}`)
      .set('Authorization', 'Bearer valid-token');

    // Assert: Check response
    expect(response.status).toBe(200);
    expect(response.body.name).toBe('John Doe');
    expect(response.body.email).toBe('john@example.com');

    // Cleanup: Remove test user
    await db.users.delete(user.id);
  });
});
```

### 3. End-to-End (E2E) Tests

**What they are:** Test complete user workflows from start to finish.

**What they test:**
- ✅ Complete user journeys (register → login → create resource → delete)
- ✅ Multiple endpoints working together
- ✅ Real database and external services
- ✅ Everything together as users experience it

**What they DON'T test:**
- ❌ Every possible edge case (too slow/expensive)

**Example:**

```javascript
// E2E test
describe('User workflow', () => {
  test('should complete full user journey', async () => {
    // 1. Register new user
    const registerRes = await request(app)
      .post('/auth/register')
      .send({
        name: 'John Doe',
        email: 'john@example.com',
        password: 'secure123'
      });
    expect(registerRes.status).toBe(201);

    // 2. Login
    const loginRes = await request(app)
      .post('/auth/login')
      .send({
        email: 'john@example.com',
        password: 'secure123'
      });
    expect(loginRes.status).toBe(200);
    const token = loginRes.body.token;

    // 3. Create post
    const postRes = await request(app)
      .post('/posts')
      .set('Authorization', `Bearer ${token}`)
      .send({
        title: 'My First Post',
        content: 'Hello World'
      });
    expect(postRes.status).toBe(201);

    // 4. Get user's posts
    const postsRes = await request(app)
      .get('/posts?user_id=' + registerRes.body.id)
      .set('Authorization', `Bearer ${token}`);
    expect(postsRes.status).toBe(200);
    expect(postsRes.body.data).toHaveLength(1);

    // 5. Delete post
    const deleteRes = await request(app)
      .delete(`/posts/${postRes.body.id}`)
      .set('Authorization', `Bearer ${token}`);
    expect(deleteRes.status).toBe(204);
  });
});
```

### 4. Load/Performance Tests

**What they are:** Test how API performs under heavy load.

**What they test:**
- ✅ Response time under normal load
- ✅ Response time under peak load
- ✅ Maximum requests per second
- ✅ How system degrades under stress
- ✅ Memory leaks and resource issues

**What they DON'T test:**
- ❌ Correctness of business logic

**Example:**

```javascript
// k6 load test
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '1m', target: 50 },   // Ramp up to 50 users
    { duration: '3m', target: 50 },   // Stay at 50 users
    { duration: '1m', target: 100 },  // Spike to 100 users
    { duration: '2m', target: 100 },  // Stay at 100 users
    { duration: '1m', target: 0 },    // Ramp down
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'], // 95% of requests < 500ms
    http_req_failed: ['rate<0.01'],   // Less than 1% failures
  },
};

export default function () {
  const res = http.get('https://api.example.com/users');
  
  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });
  
  sleep(1);
}
```

### 5. Security Tests

**What they are:** Test for security vulnerabilities.

**What they test:**
- ✅ SQL injection vulnerabilities
- ✅ XSS (Cross-Site Scripting) vulnerabilities
- ✅ Authentication bypass attempts
- ✅ Authorization flaws
- ✅ Rate limiting enforcement
- ✅ Sensitive data exposure

**What they DON'T test:**
- ❌ Business logic correctness

**Example:**

```javascript
// Security tests
describe('Security Tests', () => {
  test('should prevent SQL injection', async () => {
    const response = await request(app)
      .get('/users')
      .query({ search: "'; DROP TABLE users; --" });
    
    // Should not delete table, should handle safely
    expect(response.status).toBe(200);
  });

  test('should prevent XSS in response', async () => {
    const malicious = '<script>alert("XSS")</script>';
    
    const response = await request(app)
      .post('/posts')
      .set('Authorization', 'Bearer valid-token')
      .send({ title: malicious });
    
    expect(response.body.title).not.toContain('<script>');
  });

  test('should enforce rate limits', async () => {
    // Make 101 requests (limit is 100/hour)
    const requests = [];
    for (let i = 0; i < 101; i++) {
      requests.push(request(app).get('/users'));
    }
    
    const responses = await Promise.all(requests);
    const lastResponse = responses[100];
    
    expect(lastResponse.status).toBe(429);
  });

  test('should not expose sensitive data in errors', async () => {
    const response = await request(app)
      .get('/admin/database-config');
    
    expect(response.body).not.toMatch(/password|secret|key/i);
  });
});
```

### 6. Contract Tests

**What they are:** Test API contract between consumer and provider.

**What they test:**
- ✅ API response structure matches expectations
- ✅ Required fields are present
- ✅ Data types are correct
- ✅ Breaking changes are detected

**What they DON'T test:**
- ❌ Business logic implementation

**Example:**

```javascript
// Pact contract test
const { Pact } = require('@pact-foundation/pact');

describe('User API Contract', () => {
  const provider = new Pact({
    consumer: 'FrontendApp',
    provider: 'UserAPI',
    port: 8080,
  });

  beforeAll(() => provider.setup());
  afterAll(() => provider.finalize());

  test('should get user by ID', async () => {
    // Define expected interaction
    await provider.addInteraction({
      state: 'user with ID 42 exists',
      uponReceiving: 'a request for user 42',
      withRequest: {
        method: 'GET',
        path: '/users/42',
      },
      willRespondWith: {
        status: 200,
        headers: { 'Content-Type': 'application/json' },
        body: {
          id: 42,
          name: 'John Doe',
          email: 'john@example.com',
          // Contract: These fields must exist with these types
        },
      },
    });

    // Test against contract
    const response = await fetch('http://localhost:8080/users/42');
    const user = await response.json();

    expect(user.id).toBe(42);
    expect(typeof user.name).toBe('string');
    expect(typeof user.email).toBe('string');
  });
});
```

```javascript
        .get('/users')
        .expect(200)
        .expect('Content-Type', /json/);

      expect(response.body.data).toBeInstanceOf(Array);
      expect(response.body.pagination).toBeDefined();
    });

    test('should filter by status', async () => {
      const response = await request(app)
        .get('/users?status=active')
        .expect(200);

      response.body.data.forEach(user => {
        expect(user.status).toBe('active');
      });
    });
  });

  describe('POST /users', () => {
    test('should create new user', async () => {
      const newUser = {
        name: 'John Doe',
        email: 'john@example.com',
        password: 'secure123'
      };

      const response = await request(app)
        .post('/users')
        .send(newUser)
        .expect(201);

      expect(response.body.id).toBeDefined();
      expect(response.body.name).toBe(newUser.name);
      expect(response.body.password).toBeUndefined(); // Should not return password
    });

    test('should validate required fields', async () => {
      const response = await request(app)
        .post('/users')
        .send({ name: 'John' }) // Missing email and password
        .expect(400);

      expect(response.body.error.code).toBe('VALIDATION_ERROR');
      expect(response.body.error.details).toBeInstanceOf(Array);
    });

    test('should reject duplicate email', async () => {
      const user = {
        name: 'John Doe',
        email: 'duplicate@example.com',
        password: 'secure123'
      };

      // Create first user
      await request(app).post('/users').send(user).expect(201);

      // Try to create duplicate
      const response = await request(app)
        .post('/users')
        .send(user)
        .expect(409);

      expect(response.body.error.code).toBe('DUPLICATE_EMAIL');
    });
  });

  describe('PUT /users/:id', () => {
    test('should update user', async () => {
      // Create user first
      const createRes = await request(app)
        .post('/users')
        .send({
          name: 'John Doe',
          email: 'john@example.com',
          password: 'secure123'
        });

      const userId = createRes.body.id;

      // Update user
      const response = await request(app)
        .put(`/users/${userId}`)
        .send({ name: 'Jane Doe' })
        .expect(200);

      expect(response.body.name).toBe('Jane Doe');
    });

    test('should return 404 for non-existent user', async () => {
      await request(app)
        .put('/users/99999')
        .send({ name: 'Jane Doe' })
        .expect(404);
    });
  });

  describe('DELETE /users/:id', () => {
    test('should delete user', async () => {
      // Create user first
      const createRes = await request(app)
        .post('/users')
        .send({
          name: 'John Doe',
          email: 'temp@example.com',
          password: 'secure123'
        });

      const userId = createRes.body.id;

      // Delete user
      await request(app)
        .delete(`/users/${userId}`)
        .expect(204);

      // Verify deletion
      await request(app)
        .get(`/users/${userId}`)
        .expect(404);
    });
  });
});
```

## Load Testing

### k6 (Recommended)

```javascript
// load-test.js
import http from 'k6/http';
import { check, sleep } from 'k6';
import { Rate } from 'k6/metrics';

// Custom metrics
const errorRate = new Rate('errors');

export const options = {
  stages: [
    { duration: '30s', target: 20 },  // Ramp up to 20 users
    { duration: '1m', target: 20 },   // Stay at 20 users
    { duration: '30s', target: 50 },  // Ramp up to 50 users
    { duration: '2m', target: 50 },   // Stay at 50 users
    { duration: '30s', target: 0 },   // Ramp down
  ],
  thresholds: {
    http_req_duration: ['p(95)<500', 'p(99)<1000'], // 95% < 500ms, 99% < 1s
    http_req_failed: ['rate<0.01'],                  // Error rate < 1%
    errors: ['rate<0.1'],                            // Custom error rate < 10%
  },
};

export default function () {
  // Test GET /users
  const getRes = http.get('https://api.example.com/users');
  
  check(getRes, {
    'GET status is 200': (r) => r.status === 200,
    'GET response time < 500ms': (r) => r.timings.duration < 500,
    'GET has data array': (r) => JSON.parse(r.body).data !== undefined,
  }) || errorRate.add(1);
  
  sleep(1);
  
  // Test POST /users
  const payload = JSON.stringify({
    name: `User ${__VU}-${__ITER}`, // Virtual User - Iteration
    email: `user-${__VU}-${__ITER}@example.com`,
    password: 'test123'
  });
  
  const postRes = http.post('https://api.example.com/users', payload, {
    headers: { 'Content-Type': 'application/json' },
  });
  
  check(postRes, {
    'POST status is 201': (r) => r.status === 201,
    'POST response time < 1000ms': (r) => r.timings.duration < 1000,
    'POST returns ID': (r) => JSON.parse(r.body).id !== undefined,
  }) || errorRate.add(1);
  
  sleep(2);
}

// Custom setup/teardown
export function setup() {
  console.log('Starting load test...');
  return { startTime: new Date() };
}

export function teardown(data) {
  console.log(`Test completed. Started at: ${data.startTime}`);
}
```

```python
from zapv2 import ZAPv2
import time

# Connect to ZAP
zap = ZAPv2(proxies={'http': 'http://127.0.0.1:8080', 'https': 'http://127.0.0.1:8080'})

    print(f'Spider progress: {zap.spider.status(scan_id)}%')
```

```javascript
    '<svg onload=alert("XSS")>',

    '<iframe src="javascript:alert(\'XSS\')">',
  ];

  xssPayloads.forEach(payload => {
    test(`should prevent XSS: ${payload}`, async () => {
      const response = await request(app)
        .post('/posts')
        .set('Authorization', 'Bearer valid-token')
        .send({ title: payload, content: 'Test' });

      // Response should not contain raw script tags
      const responseText = JSON.stringify(response.body);
      expect(responseText).not.toContain('<script>');
      expect(responseText).not.toContain('onerror=');
      expect(responseText).not.toContain('javascript:');

  });
```

```javascript
    put: (url) => request(app).put(url).set('Authorization', `Bearer ${token}`),
    delete: (url) => request(app).delete(url).set('Authorization', `Bearer ${token}`),
  };
}
```

## Summary

### Testing Types Overview

| Type | What It Tests | When to Use | Speed | Cost |
|------|---------------|-------------|-------|------|
| **Unit** | Individual functions | Always | Fast ⚡ | Low 💰 |
| **Integration** | Multiple components together | Always | Medium 🚶 | Medium 💰💰 |
| **E2E** | Complete user workflows | Critical paths | Slow 🐌 | High 💰💰💰 |
| **Load** | Performance under load | Before releases | Medium 🚶 | Medium 💰💰 |
| **Security** | Vulnerabilities | Regularly | Medium 🚶 | Medium 💰💰 |
| **Contract** | API contracts | With microservices | Fast ⚡ | Low 💰 |

### Best Practices

✅ **Write tests first** (TDD) or immediately after
✅ **Test behavior, not implementation**
✅ **Keep tests independent** - Each test should run alone
✅ **Use descriptive test names** - "should return 404 when user not found"
✅ **Follow AAA pattern** - Arrange, Act, Assert
✅ **Mock external services** - Don't call real APIs in tests
✅ **Clean up after tests** - Reset database, clear caches
✅ **Run tests in CI/CD** - Automate testing
✅ **Measure coverage** - Aim for 80%+ coverage
✅ **Test error cases** - Don't just test happy paths

### What to Test

**Always Test:**
- ✅ All HTTP methods (GET, POST, PUT, DELETE)
- ✅ Status codes (200, 201, 400, 401, 404, 500)
- ✅ Request validation
- ✅ Authentication/authorization
- ✅ Error handling
- ✅ Edge cases

**Sometimes Test:**
- 📋 Rate limiting
- 📋 Pagination
- 📋 Filtering/sorting
- 📋 Performance benchmarks

**Don't Test:**
- ❌ Third-party libraries (trust them)
- ❌ Framework internals
- ❌ Configuration files
