# 10.3 Webhooks

## The Problem

You've built an API. Users want to know when something happens in your system. But how do they get notified?

**Scenario: Payment processing service**

Your service processes payments. Merchants need to know when payments succeed or fail.

**❌ Bad approach: Polling (Pull Model)**

```javascript
// Merchant's code - constantly checking for updates
setInterval(async () => {
  // Poll your API every 5 seconds
  const response = await fetch('https://api.paymentservice.com/payments/status?orderId=12345');
  const payment = await response.json();
  
  if (payment.status === 'completed') {
    console.log('Payment completed!');
    // Process the order
  }
}, 5000);

// Problems:
// ❌ Wasteful: 12 requests/minute even if nothing changed
// ❌ Delayed: Up to 5 second delay to know about status change
// ❌ Server load: 1000 merchants = 12,000 requests/minute doing nothing
// ❌ Not scalable: Faster polling = more server load
// ❌ Costs money: Both for merchants (requests) and you (processing)
```

**✅ Good approach: Webhooks (Push Model)**

```javascript
// Your service pushes updates to merchant
// When payment status changes:
async function onPaymentCompleted(payment) {
  // Send notification to merchant's webhook URL
  await fetch('https://merchant.com/webhooks/payment', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      event: 'payment.completed',
      data: {
        orderId: payment.orderId,
        amount: payment.amount,
        status: 'completed'
      }
    })
  });
}

// Merchant's webhook endpoint receives instant notification
app.post('/webhooks/payment', (req, res) => {
  const { event, data } = req.body;
  
  if (event === 'payment.completed') {
    console.log('Payment completed!');
    // Process the order immediately
  }
  
  res.sendStatus(200);
});

// Benefits:
// ✅ Instant: Merchant notified immediately when status changes
// ✅ Efficient: Only one request when something happens
// ✅ Scalable: No constant polling
// ✅ Cost-effective: Fewer requests for everyone
```

## What are Webhooks?

**Webhooks** are user-defined HTTP callbacks. When an event occurs in your system, you make an HTTP POST request to a URL provided by the user.

**Think of it as:**
- **Traditional API**: "I'll ask you when I need information" (Pull)
- **Webhook**: "Tell me when something happens" (Push)

### Pull vs Push Models

```
PULL MODEL (Traditional API Polling):
┌──────────┐                           ┌──────────┐
│  Client  │──── "Any updates?" ──────>│  Server  │
│          │<──── "No" ────────────────│          │
│          │──── "Any updates?" ──────>│          │
│          │<──── "No" ────────────────│          │
│          │──── "Any updates?" ──────>│          │
│          │<──── "Yes, here!" ────────│          │
└──────────┘                           └──────────┘
Constant polling, mostly wasted requests

PUSH MODEL (Webhooks):
┌──────────┐                           ┌──────────┐
│  Client  │ "Notify me at this URL" ─>│  Server  │
│          │                            │          │
│          │                            │ [event   │
│          │                            │ happens] │
│          │<─── "Event occurred!" ────│          │
└──────────┘                           └──────────┘
Single request when something actually happens
```

## Implementing Webhooks (Provider Side)

You're building a service and want to **send** webhooks to your users.

### Step 1: Let Users Register Webhook URLs

```javascript
// Database schema for webhook subscriptions
const webhookSchema = {
```

### Step 2: Trigger Webhooks When Events Occur

```javascript
const crypto = require('crypto');
const axios = require('axios');

// Webhook delivery service
class WebhookService {
  /**
   * Trigger webhooks for an event
   */
  async trigger(eventType, eventData, userId) {
    console.log(`Triggering webhooks for event: ${eventType}`);
    
    // Find all active webhooks subscribed to this event
    const webhooks = await db.webhooks.find({
      userId: userId,
      events: { includes: eventType },
      isActive: true
    });
    
    console.log(`Found ${webhooks.length} webhooks to notify`);
    
    // Send to all webhooks in parallel
    const deliveryPromises = webhooks.map(webhook =>
      this.deliver(webhook, eventType, eventData)
    );
    
    await Promise.allSettled(deliveryPromises);
  }
  
  /**
   * Deliver webhook payload to a single URL
   */
  async deliver(webhook, eventType, eventData) {
    const payload = {
      id: crypto.randomUUID(),
      event: eventType,
      created_at: new Date().toISOString(),
      data: eventData
    };
    
    const signature = this.sign(payload, webhook.secret);
    
    try {
      console.log(`Delivering webhook to ${webhook.url}`);
      
      const response = await axios.post(webhook.url, payload, {
        headers: {
          'Content-Type': 'application/json',
          'X-Webhook-Signature': signature,
          'X-Webhook-Event': eventType,
          'User-Agent': 'MyService-Webhooks/1.0'
        },
        timeout: 10000, // 10 second timeout
        validateStatus: (status) => status >= 200 && status < 300
      });
      
      // Update last triggered timestamp
      await db.webhooks.update(webhook.id, {
        lastTriggeredAt: new Date(),
        failureCount: 0 // Reset failure count on success
      });
      
      console.log(`Webhook delivered successfully to ${webhook.url}`);
      
      // Log successful delivery
      await db.webhookLogs.create({
        webhookId: webhook.id,
        eventType: eventType,
        status: 'success',
        statusCode: response.status,
        responseTime: response.duration
      });
      
      return { success: true };
      
    } catch (error) {
      console.error(`Webhook delivery failed to ${webhook.url}:`, error.message);
      
      // Increment failure count
      const newFailureCount = webhook.failureCount + 1;
      
      // Disable webhook after 10 consecutive failures
      const updates = {
        failureCount: newFailureCount
      };
      
      if (newFailureCount >= 10) {
        updates.isActive = false;
        console.log(`Webhook ${webhook.id} disabled after 10 failures`);
        
        // TODO: Email user about disabled webhook
      }
      
      await db.webhooks.update(webhook.id, updates);
      
      // Log failed delivery
      await db.webhookLogs.create({
        webhookId: webhook.id,
        eventType: eventType,
        status: 'failed',
        statusCode: error.response?.status,
        errorMessage: error.message
      });
      
      return { success: false, error: error.message };
    }
  }
  
  /**
   * Sign payload with HMAC for verification
   */
  sign(payload, secret) {
    const payloadString = JSON.stringify(payload);
    const hmac = crypto.createHmac('sha256', secret);
    hmac.update(payloadString);
    return hmac.digest('hex');
  }
}

const webhookService = new WebhookService();

// Usage: Trigger webhook when payment completes
app.post('/api/payments/:id/complete', async (req, res) => {
  const payment = await db.payments.findById(req.params.id);
  
  // Update payment status
  payment.status = 'completed';
  await db.payments.update(payment);
  
  // Trigger webhooks
  await webhookService.trigger('payment.completed', {
    paymentId: payment.id,
    orderId: payment.orderId,
    amount: payment.amount,
    currency: payment.currency,
    completedAt: new Date().toISOString()
  }, payment.userId);
  
  res.json({ success: true });
});
```

**How it works:**
1. User registers webhook URL via API
2. When event occurs (payment completed), you call `webhookService.trigger()`
3. Service finds all webhooks subscribed to this event
4. Makes POST request to each webhook URL
5. Includes signature for verification
6. Logs success/failure
7. Disables webhook after repeated failures

### Step 3: Implement Retry Logic

```javascript
class WebhookService {
  /**
   * Deliver with retry logic
   */
  async deliverWithRetry(webhook, eventType, eventData, attempt = 1, maxAttempts = 3) {
    try {
      await this.deliver(webhook, eventType, eventData);
      return { success: true };
      
    } catch (error) {
      console.error(`Delivery attempt ${attempt} failed:`, error.message);
      
      if (attempt < maxAttempts) {
        // Exponential backoff: wait 1s, 2s, 4s, 8s...
        const delayMs = Math.pow(2, attempt) * 1000;
        console.log(`Retrying in ${delayMs}ms...`);
        
        await new Promise(resolve => setTimeout(resolve, delayMs));
        
        return this.deliverWithRetry(webhook, eventType, eventData, attempt + 1, maxAttempts);
      }
      
      console.error(`All ${maxAttempts} delivery attempts failed`);
      return { success: false, error: error.message };
    }
  }
}
```

**Retry strategy:**
- Attempt 1: Immediate
- Attempt 2: After 2 seconds
- Attempt 3: After 4 seconds
- Give up after 3 attempts

### Step 4: Queue System for Reliability

```javascript
// Use a queue (Bull, BullMQ, etc.) for reliable webhook delivery
const Queue = require('bull');
const webhookQueue = new Queue('webhooks', 'redis://localhost:6379');

// Producer: Add webhook delivery jobs to queue
async function triggerWebhook(eventType, eventData, userId) {
  const webhooks = await db.webhooks.find({
    userId: userId,
    events: { includes: eventType },
    isActive: true
  });
  
  // Add each webhook to queue
  for (const webhook of webhooks) {
    await webhookQueue.add('deliver', {
      webhook: webhook,
      eventType: eventType,
      eventData: eventData
    }, {
      attempts: 3,        // Retry up to 3 times
      backoff: {
        type: 'exponential',
        delay: 2000       // Start with 2 second delay
      },
      removeOnComplete: true,
      removeOnFail: false // Keep failed jobs for debugging
    });
  }
}

// Consumer: Process webhook deliveries from queue
webhookQueue.process('deliver', async (job) => {
  const { webhook, eventType, eventData } = job.data;
  
  console.log(`Processing webhook delivery job ${job.id}`);
  
  await webhookService.deliver(webhook, eventType, eventData);
  
  return { delivered: true };
});

// Handle job completion
webhookQueue.on('completed', (job, result) => {
  console.log(`Job ${job.id} completed:`, result);
});

// Handle job failure (after all retries)
webhookQueue.on('failed', (job, error) => {
  console.error(`Job ${job.id} failed permanently:`, error.message);
  // TODO: Alert admin, email user
});
```

**Benefits of queue:**
- Webhooks sent asynchronously (don't block main request)
- Automatic retries with backoff
- Can handle spikes in events
- Failed deliveries preserved for debugging
- Can monitor queue health

### Step 2: Verify Webhook Signatures (Security)

**Why signatures matter:**
```javascript
// ❌ Without signature verification, anyone can fake webhooks:
// curl -X POST https://yoursite.com/webhooks/stripe \
//   -d '{"type":"payment_intent.succeeded","data":{"object":{"id":"fake_payment"}}}'
// 
// Your system processes fake payment as real!
```

**✅ With signature verification:**

```javascript
function verifyStripeSignature(payload, signature, secret) {
  // Stripe uses HMAC SHA256 signatures
  
  // Extract timestamp and signature from header
  const parts = signature.split(',');
  const timestamp = parts.find(p => p.startsWith('t=')).substring(2);
  const sig = parts.find(p => p.startsWith('v1=')).substring(3);
  
  // Check if timestamp is recent (prevent replay attacks)
  const currentTime = Math.floor(Date.now() / 1000);
  if (currentTime - timestamp > 300) { // 5 minutes
    console.error('Webhook timestamp too old');
    return false;
  }
  
  // Compute expected signature
  const signedPayload = `${timestamp}.${payload}`;
  const expectedSig = crypto
    .createHmac('sha256', secret)
    .update(signedPayload)
    .digest('hex');
  
  // Compare signatures (constant-time to prevent timing attacks)
  return crypto.timingSafeEqual(
    Buffer.from(sig),
    Buffer.from(expectedSig)
  );
}

// Usage
const STRIPE_WEBHOOK_SECRET = process.env.STRIPE_WEBHOOK_SECRET;

app.post('/webhooks/stripe', express.raw({ type: 'application/json' }), (req, res) => {
  const signature = req.headers['stripe-signature'];
  
  if (!verifyStripeSignature(req.body, signature, STRIPE_WEBHOOK_SECRET)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }
  
  // Signature valid, process webhook...
});
```

```javascript
app.post('/webhooks/stripe', async (req, res) => {
```

## Webhook Security Best Practices

### 1. Always Verify Signatures

```javascript
// ❌ Never trust webhook data without verification
app.post('/webhooks', (req, res) => {
  const event = req.body;
  processEvent(event); // Dangerous! Anyone can send fake data
});

// ✅ Always verify signature
app.post('/webhooks', (req, res) => {
  if (!verifySignature(req.body, req.headers['x-signature'], SECRET)) {
    return res.status(401).send('Invalid signature');
  }
  processEvent(req.body);
});
```

### 2. Use HTTPS Only

```javascript
// When registering webhooks:
app.post('/api/webhooks', (req, res) => {
  const url = req.body.url;
  
  // ❌ Allow HTTP
  if (!url.startsWith('http://') && !url.startsWith('https://')) {
    return res.status(400).json({ error: 'Invalid URL' });
  }
  
  // ✅ Require HTTPS
  if (!url.startsWith('https://')) {
    return res.status(400).json({
      error: 'Webhook URL must use HTTPS',
      message: 'HTTP is not secure for webhooks'
    });
  }
});
```

### 3. Prevent Replay Attacks

```javascript
function verifyWebhook(payload, signature, timestamp) {
  // Check signature
  const isValid = verifySignature(payload, signature);
  if (!isValid) return false;
  
  // Check timestamp (prevent replaying old webhooks)
  const currentTime = Math.floor(Date.now() / 1000);
  const maxAge = 300; // 5 minutes
  
  if (currentTime - timestamp > maxAge) {
    console.error(`Webhook timestamp too old: ${currentTime - timestamp}s`);
    return false;
  }
  
  return true;
}
```

### 4. Validate Event Data

```javascript
app.post('/webhooks/payment', async (req, res) => {
  const event = req.body;
  
  // Validate event structure
  if (!event.type || !event.data) {
    return res.status(400).json({ error: 'Invalid event structure' });
  }
  
  // Validate event type
  const allowedTypes = ['payment.completed', 'payment.failed', 'refund.issued'];
  if (!allowedTypes.includes(event.type)) {
    return res.status(400).json({ error: 'Unknown event type' });
  }
  
  // Validate data fields
  if (event.type === 'payment.completed') {
    if (!event.data.paymentId || !event.data.amount) {
      return res.status(400).json({ error: 'Missing required fields' });
    }
```

```javascript
// Attempt 2: 1 second later
```

### For Consumers (Receiving Webhooks)

1. **Respond quickly (< 30 seconds)**
```javascript
// ❌ Bad: Slow processing blocks response
app.post('/webhooks', async (req, res) => {
  await slowDatabaseOperation();      // 10 seconds
  await sendEmail();                   // 5 seconds
  await callExternalAPI();             // 15 seconds
  res.sendStatus(200);                 // 30 seconds total - might timeout!
});

// ✅ Good: Acknowledge immediately, process async
app.post('/webhooks', async (req, res) => {
  await queue.add('process-webhook', req.body);
  res.sendStatus(200);  // < 1 second
});
```

2. **Always verify signatures**
```javascript
// Never skip signature verification
if (!verifySignature(req.body, req.headers['x-signature'], SECRET)) {
  return res.status(401).json({ error: 'Invalid signature' });
}
```

3. **Handle duplicates (idempotency)**
```javascript
// Use event ID to detect duplicates
const eventId = req.body.id;
if (await isAlreadyProcessed(eventId)) {
  return res.sendStatus(200); // Already processed, acknowledge
}
await markAsProcessed(eventId);
```

4. **Log everything**

## Event-Driven Architecture

Webhooks enable **event-driven architecture** where systems react to events rather than polling.

### Example: E-commerce Order Flow

```javascript
// Traditional approach (tightly coupled):
async function createOrder(orderData) {
  const order = await db.orders.create(orderData);
  await inventoryService.reserveStock(order);        // Coupled
  await paymentService.charge(order);                // Coupled
  await shippingService.createShipment(order);       // Coupled
  await emailService.sendConfirmation(order);        // Coupled
  await analyticsService.trackPurchase(order);       // Coupled
  return order;
}

// Event-driven approach (loosely coupled):
async function createOrder(orderData) {
  const order = await db.orders.create(orderData);
  
  // Emit event - other services listen
  await eventBus.publish('order.created', order);
  
  return order; // Fast response, other operations happen async
}

// Inventory service listens for events
eventBus.subscribe('order.created', async (order) => {
  await reserveStock(order);
  await eventBus.publish('stock.reserved', order);
});

// Payment service listens for events
eventBus.subscribe('stock.reserved', async (order) => {
  await chargeCustomer(order);
  await eventBus.publish('payment.completed', order);
});

// Shipping service listens for events
eventBus.subscribe('payment.completed', async (order) => {
  await createShipment(order);
  await eventBus.publish('shipment.created', order);
});

// Email service listens for events  
eventBus.subscribe('shipment.created', async (order) => {
  await sendShippingConfirmation(order);
});
```

**Benefits of event-driven:**
- Services are loosely coupled
- Easy to add new services (just subscribe to events)
- Failures are isolated (payment failure doesn't break stock reservation)
- Can process events async (faster response to user)

## Comparison: Webhooks vs Alternatives

| Method | How it works | Pros | Cons | Use Case |
|--------|-------------|------|------|----------|
| **Polling** | Client asks "any updates?" repeatedly | Simple to implement | Wasteful, delayed updates, high server load | Infrequent updates, simple apps |
| **Webhooks** | Server pushes updates to client URL | Instant, efficient, scalable | Requires public URL, harder to debug | Real-time notifications, integrations |
| **WebSocket** | Persistent bidirectional connection | Real-time, bidirectional | Complex, stateful, hard to scale | Chat, live dashboards, gaming |
| **Server-Sent Events** | Server streams updates to client | Simple, built into browsers | Server → client only, connection kept open | Live feeds, notifications |
| **Message Queue** | Producer → Queue → Consumer | Reliable, async, scalable | Infrastructure overhead | Internal services, high volume |

## Key Takeaways

1. **Webhooks are HTTP callbacks** - "Call me when something happens"

2. **Push vs Pull**: Webhooks push updates (efficient) vs polling pulls updates (wasteful)

3. **Security is critical**: Always verify signatures, use HTTPS, validate data

4. **Respond quickly**: Acknowledge webhook < 30 seconds, process asynchronously

5. **Idempotency matters**: Webhooks might be delivered multiple times, handle duplicates

6. **Retry logic**: Implement exponential backoff, give up after reasonable attempts

7. **Queue for reliability**: Use message queues for robust webhook delivery

8. **Monitor everything**: Log webhook deliveries, track failures, alert on issues

9. **Enable event-driven architecture**: Webhooks facilitate loosely coupled systems

10. **Choose the right tool**:
    - Webhooks: External integrations, cross-service updates
    - WebSocket: Real-time bidirectional (chat, gaming)
    - Polling: Simple apps, infrequent updates
    - Message Queue: Internal services, high reliability
