# API Performance Optimization - Making Your API Fast

## Introduction

**Performance optimization** is about making your API fast, scalable, and efficient. A slow API frustrates users and costs money.

**Key principle:** Performance is a feature, not an afterthought.

---

## Performance Metrics - What to Measure

### Key Metrics

```
┌─────────────────────────────────────────────┐
│  Request Journey                            │
├─────────────────────────────────────────────┤
│                                             │
│  Client → Network → Server → Database      │
│           ↓         ↓         ↓             │
│         50ms      100ms     150ms           │
│                                             │
│  Total Response Time: 300ms                 │
└─────────────────────────────────────────────┘
```

**What to measure:**
- **Response Time** - How long requests take (p50, p95, p99)
- **Throughput** - Requests per second (RPS)
- **Error Rate** - Percentage of failed requests
- **CPU Usage** - Server processor utilization
- **Memory Usage** - RAM consumption
- **Database Query Time** - How long DB operations take

**Performance targets:**
- ✅ **p50 < 100ms** - Half of requests under 100ms
- ✅ **p95 < 500ms** - 95% under 500ms
- ✅ **p99 < 1000ms** - 99% under 1 second
- ✅ **Error rate < 1%** - Less than 1% failures
- ✅ **Throughput > 1000 RPS** - Handle 1000+ requests/second

## 1. Database Indexing

### The Problem

```javascript
// ❌ Slow: Full table scan
// Query takes 2000ms on 1 million users
app.get('/users', async (req, res) => {
  const { status } = req.query;
  
  // No index on 'status' column
  const users = await db.users.find({ status });
  res.json({ data: users });
});

// Database scans ALL 1 million rows to find active users
```

### The Solution: Add Indexes

```javascript
// ✅ Fast: Uses index
// Query takes 10ms on 1 million users

// Create index on status column
db.users.createIndex({ status: 1 });

app.get('/users', async (req, res) => {
  const { status } = req.query;
  
  // Now uses index - super fast!
  const users = await db.users.find({ status });
  res.json({ data: users });
});
```

### Index Types

#### 1. Single Field Index

```javascript
// Index on email for fast lookups
db.users.createIndex({ email: 1 });

// Fast query
const user = await db.users.findOne({ email: 'john@example.com' });
// Uses index: 5ms
```

#### 2. Compound Index

```javascript
// Index on multiple fields (order matters!)
db.users.createIndex({ status: 1, created_at: -1 });

// Fast query - uses compound index
const activeUsers = await db.users.find({ status: 'active' })
  .sort({ created_at: -1 });
// Uses index: 8ms

// Also fast - uses partial index
const users = await db.users.find({ status: 'active' });
// Uses index: 5ms

// ❌ Slow - doesn't use index (wrong field order)
const recent = await db.users.find().sort({ created_at: -1 });
// No index: 500ms
```

#### 3. Unique Index

```javascript
// Prevent duplicate emails
db.users.createIndex({ email: 1 }, { unique: true });

// Automatically enforced
try {
  await db.users.insert({ email: 'existing@example.com' });
} catch (error) {
  // Error: Duplicate key
}
```

#### 4. Text Index (Full-Text Search)

```javascript
// Index for text search
db.posts.createIndex({ 
  title: 'text', 
  content: 'text' 
});

// Fast text search
const results = await db.posts.find({
  $text: { $search: 'javascript tutorial' }
});
```

#### 5. Partial Index (Filtered)

```javascript
// Only index active users
db.users.createIndex(
  { email: 1 },
  { partialFilterExpression: { status: 'active' } }
);

// Uses partial index
const user = await db.users.findOne({ 
  email: 'john@example.com',
  status: 'active' 
});
```

### SQL Indexing

```sql
-- Create index
CREATE INDEX idx_users_status ON users(status);

-- Compound index
CREATE INDEX idx_users_status_created ON users(status, created_at DESC);

-- Unique index
CREATE UNIQUE INDEX idx_users_email ON users(email);

-- Partial index (PostgreSQL)
CREATE INDEX idx_active_users_email ON users(email)
WHERE status = 'active';

-- Check if index is used
EXPLAIN ANALYZE
SELECT * FROM users WHERE status = 'active';

-- Output shows "Index Scan using idx_users_status" ✅
```

### Index Best Practices

```javascript
// ✅ Index columns used in WHERE clauses
db.users.createIndex({ status: 1 });
// SELECT * FROM users WHERE status = 'active'

// ✅ Index columns used in sorting
db.users.createIndex({ created_at: -1 });
// SELECT * FROM users ORDER BY created_at DESC

// ✅ Index foreign keys
db.posts.createIndex({ user_id: 1 });
// SELECT * FROM posts WHERE user_id = 42

// ✅ Compound indexes for common queries
db.posts.createIndex({ user_id: 1, status: 1, created_at: -1 });
// SELECT * FROM posts WHERE user_id = 42 AND status = 'published' ORDER BY created_at DESC

// ❌ Don't over-index
// Each index slows down writes (INSERT, UPDATE, DELETE)
// Rule: Only index columns you actually query

// ❌ Don't index low-cardinality columns alone
// Bad: INDEX on boolean (true/false)
// Good: COMPOUND INDEX with boolean + high-cardinality column
db.users.createIndex({ is_admin: 1 }); // ❌ Bad (only 2 values)
db.users.createIndex({ status: 1, created_at: -1 }); // ✅ Good
```

## 2. The N+1 Query Problem

### The Problem

```javascript
// ❌ N+1 Problem: 1 query for users + N queries for posts
app.get('/users', async (req, res) => {
  // 1 query: Get all users
  const users = await db.users.find();
  
  // N queries: Get posts for each user (if 100 users = 100 queries!)
  const usersWithPosts = await Promise.all(
    users.map(async (user) => {
      const posts = await db.posts.find({ user_id: user.id });
      return { ...user, posts };
    })
  );
  
  res.json({ data: usersWithPosts });
});

// Total: 101 database queries! 😱
```

### Solution 1: Eager Loading (Join)

```javascript
// ✅ Solution: 1 query with JOIN
app.get('/users', async (req, res) => {
  const users = await db.query(`
    SELECT 
      users.*,
      JSON_AGG(posts.*) as posts
    FROM users
    LEFT JOIN posts ON posts.user_id = users.id
    GROUP BY users.id
  `);
  
  res.json({ data: users });
});

// Total: 1 database query! ⚡
```

### Solution 2: Batch Loading

```javascript
// ✅ Solution: 2 queries total
app.get('/users', async (req, res) => {
  // Query 1: Get all users
  const users = await db.users.find();
  const userIds = users.map(u => u.id);
  
  // Query 2: Get all posts for these users (IN clause)
  const posts = await db.posts.find({ 
    user_id: { $in: userIds } 
  });
  
  // Organize posts by user_id
  const postsByUser = posts.reduce((acc, post) => {
    if (!acc[post.user_id]) acc[post.user_id] = [];
    acc[post.user_id].push(post);
    return acc;
  }, {});
  
  // Combine users with their posts
  const usersWithPosts = users.map(user => ({
    ...user,
    posts: postsByUser[user.id] || []
  }));
  
  res.json({ data: usersWithPosts });
});

// Total: 2 database queries! ✅
```

### Solution 3: DataLoader (Batching + Caching)

```javascript
const DataLoader = require('dataloader');

// Create a data loader
const postLoader = new DataLoader(async (userIds) => {
  // Batch: Load posts for all requested users
  const posts = await db.posts.find({
    user_id: { $in: userIds }
  });
  
  // Organize by user_id
  const postsByUser = userIds.map(userId =>
    posts.filter(post => post.user_id === userId)
  );
  
  return postsByUser;
});

// Usage
app.get('/users', async (req, res) => {
  const users = await db.users.find();
  
  // Load posts using DataLoader (automatically batches + caches)
  const usersWithPosts = await Promise.all(
    users.map(async (user) => ({
      ...user,
      posts: await postLoader.load(user.id)
    }))
  );
  
  res.json({ data: usersWithPosts });
});

// Total: 2 queries, with automatic batching and caching! 🚀
```

## 3. Response Compression

### Without Compression

```javascript
// ❌ Sending 500KB of JSON uncompressed
app.get('/users', async (req, res) => {
  const users = await db.users.find();
  res.json({ data: users });
});

// Response size: 500 KB
// Transfer time (3G): 500KB / 50KB/s = 10 seconds 😱
```

### With Compression

```javascript
const compression = require('compression');

// ✅ Enable compression
app.use(compression({
  level: 6,           // Compression level (0-9, default: 6)
  threshold: 1024,    // Only compress responses > 1KB
  filter: (req, res) => {
    // Don't compress images/videos
    if (req.headers['x-no-compression']) {
      return false;
    }
    return compression.filter(req, res);
  }
}));

app.get('/users', async (req, res) => {
  const users = await db.users.find();
  res.json({ data: users });
});

// Response size: 50 KB (10x smaller!)
// Transfer time (3G): 50KB / 50KB/s = 1 second ⚡
```

### Compression Algorithms

```javascript
// 1. gzip (default) - Good compression, fast
// Content-Encoding: gzip
// Compression ratio: ~70-80% reduction

// 2. brotli - Better compression, slower
const shrinkRay = require('shrink-ray-current');

app.use(shrinkRay({
  brotli: {
    quality: 11  // 0-11 (11 = best compression)
  }
}));

// Compression ratio: ~80-85% reduction

// 3. deflate - Older, less common
// Same as gzip but different format
```

### Manual Compression

```javascript
const zlib = require('zlib');
const { promisify } = require('util');

const gzip = promisify(zlib.gzip);
const brotliCompress = promisify(zlib.brotliCompress);

app.get('/users', async (req, res) => {
  const users = await db.users.find();
  const json = JSON.stringify({ data: users });
  
  const acceptEncoding = req.headers['accept-encoding'] || '';
  
  if (acceptEncoding.includes('br')) {
    // Brotli compression
    const compressed = await brotliCompress(json);
    res.set('Content-Encoding', 'br');
    res.send(compressed);
  } else if (acceptEncoding.includes('gzip')) {
    // Gzip compression
    const compressed = await gzip(json);
    res.set('Content-Encoding', 'gzip');
    res.send(compressed);
  } else {
    // No compression
    res.json({ data: users });
  }
});
```

## 4. Database Connection Pooling

### The Problem

```javascript
// ❌ Creating new connection for each request
app.get('/users', async (req, res) => {
  // Opens new database connection (100ms overhead)
  const db = await connectToDatabase();
  const users = await db.users.find();
  await db.close(); // Closes connection
  
  res.json({ data: users });
});

// Every request wastes 100ms opening/closing connections!
```

### Solution: Connection Pool

```javascript
const { Pool } = require('pg');

// ✅ Create connection pool (reuse connections)
const pool = new Pool({
  host: 'localhost',
  database: 'mydb',
  user: 'user',
  password: 'pass',
  max: 20,              // Maximum connections in pool
  min: 5,               // Minimum connections to keep open
  idleTimeoutMillis: 30000,  // Close idle connections after 30s
  connectionTimeoutMillis: 2000,  // Fail after 2s if no connection available
});

app.get('/users', async (req, res) => {
  // Reuses existing connection from pool (instant!)
  const client = await pool.connect();
  
  try {
    const result = await client.query('SELECT * FROM users');
    res.json({ data: result.rows });
  } finally {
    client.release(); // Return connection to pool
  }
});

// Connections are reused - much faster! ⚡
```

### MongoDB Connection Pooling

```javascript
const { MongoClient } = require('mongodb');

const client = new MongoClient(uri, {
  maxPoolSize: 50,      // Max connections
  minPoolSize: 10,      // Min connections
  maxIdleTimeMS: 30000  // Idle timeout
});

await client.connect();
const db = client.db('mydb');

app.get('/users', async (req, res) => {
  // Uses connection from pool automatically
  const users = await db.collection('users').find().toArray();
  res.json({ data: users });
});
```

## 5. Async Operations and Parallelization

### Sequential vs Parallel

```javascript
// ❌ Sequential: 600ms total
app.get('/dashboard', async (req, res) => {
  const users = await fetchUsers();      // 200ms
  const posts = await fetchPosts();      // 200ms
  const comments = await fetchComments(); // 200ms
  
  res.json({ users, posts, comments });
});
// Total: 200 + 200 + 200 = 600ms

// ✅ Parallel: 200ms total
app.get('/dashboard', async (req, res) => {
  const [users, posts, comments] = await Promise.all([
    fetchUsers(),      // 200ms
    fetchPosts(),      // 200ms (parallel)
    fetchComments()    // 200ms (parallel)
  ]);
  
  res.json({ users, posts, comments });
});
// Total: max(200, 200, 200) = 200ms ⚡
```

### Real Example

```javascript
// ❌ Slow: Sequential queries (450ms)
app.get('/users/:id', async (req, res) => {
  const user = await db.users.findOne({ id: req.params.id });        // 50ms
  const posts = await db.posts.find({ user_id: user.id });           // 100ms
  const followers = await db.follows.count({ following_id: user.id }); // 100ms
  const following = await db.follows.count({ follower_id: user.id }); // 100ms
  const stats = await db.stats.findOne({ user_id: user.id });        // 100ms
  
  res.json({ user, posts, followers, following, stats });
});

// ✅ Fast: Parallel queries (100ms)
app.get('/users/:id', async (req, res) => {
  const user = await db.users.findOne({ id: req.params.id }); // 50ms
  
  // Run these in parallel
  const [posts, followers, following, stats] = await Promise.all([
    db.posts.find({ user_id: user.id }),
    db.follows.count({ following_id: user.id }),
    db.follows.count({ follower_id: user.id }),
    db.stats.findOne({ user_id: user.id })
  ]);
  
  res.json({ user, posts, followers, following, stats });
});
// Total: 50 + max(100, 100, 100, 100) = 150ms ⚡
```

### Promise.allSettled (Handle Failures)

```javascript
// ✅ Don't let one failure break everything
app.get('/dashboard', async (req, res) => {
  const results = await Promise.allSettled([
    fetchUsers(),      // Might fail
    fetchPosts(),      // Might fail
    fetchComments()    // Might fail
  ]);
  
  const [usersResult, postsResult, commentsResult] = results;
  
  const data = {
    users: usersResult.status === 'fulfilled' ? usersResult.value : [],
    posts: postsResult.status === 'fulfilled' ? postsResult.value : [],
    comments: commentsResult.status === 'fulfilled' ? commentsResult.value : []
  };
  
  res.json(data);
});
// Even if one query fails, others still return data ✅
```

## 6. Pagination and Field Selection

### Pagination Reduces Load

```javascript
// ❌ Bad: Returns ALL users (1 million rows!)
app.get('/users', async (req, res) => {
  const users = await db.users.find();
  res.json({ data: users }); // 500MB response!
});

// ✅ Good: Returns 20 users at a time
app.get('/users', async (req, res) => {
  const page = parseInt(req.query.page) || 1;
  const limit = parseInt(req.query.limit) || 20;
  const skip = (page - 1) * limit;
  
  const users = await db.users.find()
    .limit(limit)
    .skip(skip);
  
  res.json({ data: users, page, limit });
});
```

### Field Selection (Sparse Fieldsets)

```javascript
// ❌ Bad: Returns all fields
app.get('/users', async (req, res) => {
  const users = await db.users.find();
  // Returns: id, name, email, password_hash, bio, avatar, created_at, updated_at, ...
  res.json({ data: users });
});

// ✅ Good: Returns only requested fields
app.get('/users', async (req, res) => {
  const fields = req.query.fields || 'id,name,email';
  const projection = fields.split(',').reduce((acc, field) => {
    acc[field] = 1;
    return acc;
  }, {});
  
  const users = await db.users.find({}, projection);
  // Returns only: id, name, email
  res.json({ data: users });
});

// Usage: GET /users?fields=id,name
```

## 7. Caching Strategies

### Multi-Layer Caching

```
┌─────────────────────────────────────┐
│  Client Browser Cache               │  ← 0ms (instant)
├─────────────────────────────────────┤
│  CDN Cache (CloudFlare, Akamai)    │  ← 10ms
├─────────────────────────────────────┤
│  Application Cache (Redis)          │  ← 5ms
├─────────────────────────────────────┤
│  Database Query Cache               │  ← 20ms
├─────────────────────────────────────┤
│  Database (Disk)                    │  ← 100ms
└─────────────────────────────────────┘
```

### In-Memory Cache (Node.js)

```javascript
const cache = new Map();

app.get('/users/:id', async (req, res) => {
  const userId = req.params.id;
  const cacheKey = `user:${userId}`;
  
  // Check cache first
  if (cache.has(cacheKey)) {
    console.log('Cache HIT');
    return res.json(cache.get(cacheKey));
  }
  
  // Cache miss - fetch from database
  console.log('Cache MISS');
  const user = await db.users.findOne({ id: userId });
  
  // Store in cache for 5 minutes
  cache.set(cacheKey, user);
  setTimeout(() => cache.delete(cacheKey), 5 * 60 * 1000);
  
  res.json(user);
});

// First request: 100ms (database)
// Subsequent requests: 1ms (cache) ⚡
```

### Redis Cache

```javascript
const Redis = require('ioredis');
const redis = new Redis();

app.get('/users/:id', async (req, res) => {
  const userId = req.params.id;
  const cacheKey = `user:${userId}`;
  
  // Try cache
  const cached = await redis.get(cacheKey);
  if (cached) {
    console.log('Cache HIT');
    return res.json(JSON.parse(cached));
  }
  
  // Cache miss
  console.log('Cache MISS');
  const user = await db.users.findOne({ id: userId });
  
  // Cache for 5 minutes
  await redis.setex(cacheKey, 300, JSON.stringify(user));
  
  res.json(user);
});
```

### Cache Middleware

```javascript
function cacheMiddleware(ttl = 300) {
  return async (req, res, next) => {
    const cacheKey = `cache:${req.originalUrl}`;
    
    // Try cache
    const cached = await redis.get(cacheKey);
    if (cached) {
      res.set('X-Cache', 'HIT');
      return res.json(JSON.parse(cached));
    }
    
    // Override res.json to cache response
    const originalJson = res.json.bind(res);
    res.json = (data) => {
      redis.setex(cacheKey, ttl, JSON.stringify(data));
      res.set('X-Cache', 'MISS');
      return originalJson(data);
    };
    
    next();
  };
}

// Usage
app.get('/users', cacheMiddleware(300), async (req, res) => {
  const users = await db.users.find();
  res.json({ data: users });
});
```

## 8. Query Optimization

### Use EXPLAIN to Analyze Queries

```sql
-- Check query performance
EXPLAIN ANALYZE
SELECT * FROM users WHERE status = 'active';

-- Good output (uses index):
-- Index Scan using idx_users_status on users (cost=0.43..8.45 rows=1 width=72)
-- Execution Time: 0.123 ms ✅

-- Bad output (full table scan):
-- Seq Scan on users (cost=0.00..1693.00 rows=50000 width=72)
-- Execution Time: 45.678 ms ❌
```

### Avoid SELECT *

```javascript
// ❌ Bad: Fetches all columns (wasteful)
const users = await db.query('SELECT * FROM users');

// ✅ Good: Fetch only needed columns
const users = await db.query('SELECT id, name, email FROM users');
```

### Use WHERE Conditions Effectively

```javascript
// ❌ Bad: Filters in application (fetches everything)
const users = await db.users.find();
const activeUsers = users.filter(u => u.status === 'active');

// ✅ Good: Filters in database
const activeUsers = await db.users.find({ status: 'active' });
```

### Limit Results

```javascript
// ❌ Bad: Fetches all results
const recentPosts = await db.posts.find()
  .sort({ created_at: -1 });

// ✅ Good: Limits to what you need
const recentPosts = await db.posts.find()
  .sort({ created_at: -1 })
  .limit(10);
```

## 9. Streaming Large Responses

### Problem: Large Responses

```javascript
// ❌ Bad: Loads everything into memory
app.get('/export', async (req, res) => {
  const users = await db.users.find(); // 1 million users = 500MB in memory!
  
  const csv = users.map(u => `${u.id},${u.name},${u.email}`).join('\n');
  res.send(csv); // Server crashes with 500MB response!
});
```

### Solution: Stream

```javascript
// ✅ Good: Streams data (constant memory usage)
const { Transform } = require('stream');

app.get('/export', async (req, res) => {
  res.setHeader('Content-Type', 'text/csv');
  res.setHeader('Content-Disposition', 'attachment; filename=users.csv');
  
  // Stream from database
  const cursor = db.users.find().cursor();
  
  // Transform to CSV
  const csvTransform = new Transform({
    transform(user, encoding, callback) {
      const csv = `${user.id},${user.name},${user.email}\n`;
      callback(null, csv);
    }
  });
  
  // Pipe: Database → Transform → Response
  cursor.pipe(csvTransform).pipe(res);
});

// Memory usage: Constant (only processes one row at a time) ⚡
```

## 10. Background Jobs

### Problem: Slow Operations Block Response

```javascript
// ❌ Bad: Client waits for email to be sent
app.post('/users', async (req, res) => {
  const user = await db.users.create(req.body);
  
  // This takes 3 seconds!
  await sendWelcomeEmail(user.email);
  
  res.status(201).json(user);
});
// Response time: 3050ms (3 seconds!) 😱
```

### Solution: Background Queue

```javascript
const Queue = require('bull');
const emailQueue = new Queue('email', 'redis://localhost:6379');

// ✅ Good: Queue the email, respond immediately
app.post('/users', async (req, res) => {
  const user = await db.users.create(req.body);
  
  // Add to queue (instant)
  await emailQueue.add({ email: user.email });
  
  res.status(201).json(user);
});
// Response time: 50ms ⚡

// Process emails in background
emailQueue.process(async (job) => {
  await sendWelcomeEmail(job.data.email);
});
```

## 11. HTTP/2 and Keep-Alive

### Enable HTTP/2

```javascript
const http2 = require('http2');
const fs = require('fs');

// HTTP/2 server
const server = http2.createSecureServer({
  key: fs.readFileSync('server.key'),

    'content-type': 'application/json',

server.listen(443);

// Benefits:
```

## Performance Optimization Checklist

### Database

- ✅ **Add indexes** on columns used in WHERE, ORDER BY, JOIN
- ✅ **Fix N+1 queries** - Use joins or batch loading
- ✅ **Use connection pooling** - Reuse database connections
- ✅ **Optimize queries** - Use EXPLAIN, avoid SELECT *
- ✅ **Add database read replicas** for read-heavy workloads

### API Layer

- ✅ **Enable compression** - gzip/brotli for responses
- ✅ **Implement caching** - Redis, in-memory, HTTP caching
- ✅ **Paginate results** - Don't return everything at once
- ✅ **Support field selection** - Let clients request specific fields

## Real-World Example: Complete Optimization

### Before Optimization

```javascript
// ❌ Slow API (response time: 2500ms)
app.get('/posts', async (req, res) => {

  const posts = await db.posts.find();

    post.author = await db.users.findOne({ id: post.user_id });

  // No compression
  // No indexes
```

### After Optimization

```javascript
// ✅ Fast API (response time: 50ms)
const compression = require('compression');
const Redis = require('ioredis');
const redis = new Redis();

// 1. Enable compression
app.use(compression());

// 2. Add database indexes
db.posts.createIndex({ status: 1, created_at: -1 });
db.posts.createIndex({ user_id: 1 });
db.comments.createIndex({ post_id: 1 });

app.get('/posts', async (req, res) => {
  // 3. Pagination
  const page = parseInt(req.query.page) || 1;
  const limit = Math.min(parseInt(req.query.limit) || 20, 100);
  const skip = (page - 1) * limit;
  
  // 4. Cache key
  const cacheKey = `posts:page:${page}:limit:${limit}`;
  
  // 5. Check cache
  const cached = await redis.get(cacheKey);
  if (cached) {
    res.set('X-Cache', 'HIT');
    return res.json(JSON.parse(cached));
  }
  
  // 6. Efficient query with limit
  const posts = await db.posts.find({ status: 'published' })
    .sort({ created_at: -1 })
    .limit(limit)
    .skip(skip);
  
  // 7. Fix N+1: Batch load authors and comments
  const userIds = [...new Set(posts.map(p => p.user_id))];
  const postIds = posts.map(p => p.id);
  
  const [authors, comments] = await Promise.all([
    db.users.find({ id: { $in: userIds } }),
    db.comments.find({ post_id: { $in: postIds } })
  ]);
  
  // 8. Organize data
  const authorsById = Object.fromEntries(authors.map(a => [a.id, a]));
  const commentsByPost = comments.reduce((acc, c) => {
    if (!acc[c.post_id]) acc[c.post_id] = [];
    acc[c.post_id].push(c);
    return acc;
  }, {});
  
  const postsWithData = posts.map(post => ({
    ...post,
    author: authorsById[post.user_id],
    comments: commentsByPost[post.id] || []
  }));
  
  const response = { data: postsWithData, page, limit };
  
  // 9. Cache for 5 minutes
  await redis.setex(cacheKey, 300, JSON.stringify(response));
  res.set('X-Cache', 'MISS');
  
  // 10. HTTP caching headers
  res.set('Cache-Control', 'public, max-age=60');
  
  res.json(response);
});

// Performance improvements:
// ✅ 2500ms → 50ms (50x faster!)
// ✅ 101 queries → 3 queries
// ✅ 500KB → 50KB (10x smaller with compression)
// ✅ Cached responses serve in 5ms
```

❌ **No connection pooling** - Create new connections per request
