# Monitoring and Logging - Observability for REST APIs

## The Problem: Flying Blind Without Monitoring

### What Happens Without Proper Logging?

```
User: "Your API is broken!"
Developer: "What endpoint? What data? When did this happen?"
User: "I don't know, it just doesn't work!"
Developer: 🤷 *No logs, no idea what went wrong*
```

**Without monitoring and logging:**
- 🔴 Can't debug production issues
- 🔴 Don't know when things break
- 🔴 Can't identify performance bottlenecks
- 🔴 Don't understand user behavior
- 🔴 Security breaches go undetected
- 🔴 No visibility into system health

---

## Why Monitoring and Logging Matter

### The Three Pillars of Observability

```
1. LOGS       → What happened? (events)
2. METRICS    → How much/many? (numbers)
3. TRACES     → Where did it go? (request flow)

Together = Complete system visibility
```

### Business Impact

```
Without monitoring:
- Mean Time to Detect (MTTD): 4 hours
- Mean Time to Resolve (MTTR): 8 hours
- Total downtime: 12 hours
- Revenue lost: $120,000

With monitoring:
- MTTD: 2 minutes (alert fires)
- MTTR: 30 minutes (logs show root cause)
- Total downtime: 32 minutes
- Revenue lost: $3,200

Savings: $116,800 per incident!
```

## 1. Request/Response Logging

### What to Log

#### Basic Request Information

```javascript
const express = require('express');
const app = express();

// Request logging middleware
app.use((req, res, next) => {
  const startTime = Date.now();
  
  // Log request
  console.log({
    timestamp: new Date().toISOString(),
    type: 'request',
    method: req.method,
    url: req.url,
    path: req.path,
    query: req.query,
    ip: req.ip,
    userAgent: req.get('user-agent'),
    correlationId: req.headers['x-correlation-id'] || generateId(),
    userId: req.user?.id || null
  });
  
  // Capture response
  const originalSend = res.send;
  res.send = function(data) {
    const duration = Date.now() - startTime;
    
    // Log response
    console.log({
      timestamp: new Date().toISOString(),
      type: 'response',
      method: req.method,
      url: req.url,
      statusCode: res.statusCode,
      duration: duration,
      contentLength: data?.length || 0,
      correlationId: req.headers['x-correlation-id']
    });
    
    originalSend.call(this, data);
  };
  
  next();
});
```

### Structured Logging

```javascript
// ❌ Bad: Unstructured logs (hard to search/parse)
console.log('User 42 accessed /api/users at 10:30:45');

// ✅ Good: Structured logs (JSON, easy to search)
const winston = require('winston');

const logger = winston.createLogger({
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })
  ]
});

logger.info('User accessed endpoint', {
  userId: 42,
  endpoint: '/api/users',
  timestamp: new Date().toISOString(),
  ip: '203.0.113.42',
  responseTime: 111
});
```

**Output:**
```json
{
  "level": "info",
  "message": "User accessed endpoint",
  "userId": 42,
```

### Correlation IDs

**Problem:** Track a single request across multiple services

```
Client Request
  ↓ [correlation-id: abc123]
API Gateway
  ↓ [correlation-id: abc123]
Auth Service
  ↓ [correlation-id: abc123]
Database
```

**Implementation:**

```javascript
const { v4: uuidv4 } = require('uuid');

// Add correlation ID middleware
app.use((req, res, next) => {
  // Use existing correlation ID or generate new one
  const correlationId = req.headers['x-correlation-id'] || uuidv4();
  
  // Attach to request
  req.correlationId = correlationId;
  
  // Add to response headers
  res.setHeader('X-Correlation-ID', correlationId);
  
  next();
});

// Use in all logs
logger.info('Processing request', {
  correlationId: req.correlationId,
  endpoint: req.path
});

// Pass to downstream services
const response = await fetch('https://api.downstream.com/data', {
  headers: {
    'X-Correlation-ID': req.correlationId
  }
});
```

**Finding issues:**
```bash
# All logs for single request across all services
grep "abc123" *.log

# Or with structured logging
jq '.correlationId == "abc123"' logs.json
```

## 2. Performance Metrics

### What to Measure

#### Response Time Metrics

```javascript
const prometheus = require('prom-client');

// Create histogram for response times
const httpRequestDuration = new prometheus.Histogram({
  name: 'http_request_duration_seconds',
  help: 'Duration of HTTP requests in seconds',
  labelNames: ['method', 'route', 'status_code'],
  buckets: [0.1, 0.3, 0.5, 0.7, 1, 3, 5, 7, 10] // seconds
});

app.use((req, res, next) => {
  const start = Date.now();
  
  res.on('finish', () => {
    const duration = (Date.now() - start) / 1000; // Convert to seconds
    
    httpRequestDuration
      .labels(req.method, req.route?.path || req.path, res.statusCode)
      .observe(duration);
  });
  
  next();
});

// Query metrics
// p50 (median): 50% of requests faster than this
// p95: 95% of requests faster than this
// p99: 99% of requests faster than this

/*
GET /api/users
  p50: 120ms   ← Half of requests under 120ms
  p95: 450ms   ← 95% under 450ms
  p99: 1200ms  ← 1% of requests take over 1.2s
*/
```

#### Throughput Metrics

```javascript
// Requests per second
const httpRequestsTotal = new prometheus.Counter({
  name: 'http_requests_total',
  help: 'Total number of HTTP requests',
  labelNames: ['method', 'route', 'status_code']
});

app.use((req, res, next) => {
  res.on('finish', () => {
    httpRequestsTotal
      .labels(req.method, req.route?.path || req.path, res.statusCode)
      .inc();
  });
  next();
});

// Query: rate(http_requests_total[5m])
// Result: 450 requests/second
```

#### Error Rate Metrics

```javascript
// Track errors
const httpRequestErrors = new prometheus.Counter({
  name: 'http_request_errors_total',
  help: 'Total number of HTTP errors',
  labelNames: ['method', 'route', 'status_code']
});

app.use((req, res, next) => {
  res.on('finish', () => {
    if (res.statusCode >= 400) {
      httpRequestErrors
        .labels(req.method, req.route?.path || req.path, res.statusCode)
        .inc();
    }
  });
  next();
});

// Calculate error rate
// Error rate = (errors / total requests) * 100
// 5xx errors / total * 100 = 0.5% error rate
```

#### Database Metrics

```javascript
// Query duration
const dbQueryDuration = new prometheus.Histogram({
  name: 'db_query_duration_seconds',
  help: 'Database query duration',
  labelNames: ['operation', 'table'],
  buckets: [0.01, 0.05, 0.1, 0.5, 1, 2, 5]
});

// Wrap database queries
async function trackQuery(operation, table, queryFn) {
  const start = Date.now();
  
  try {
    const result = await queryFn();
    
    const duration = (Date.now() - start) / 1000;
    dbQueryDuration.labels(operation, table).observe(duration);
    
    return result;
  } catch (error) {
    dbQueryErrors.labels(operation, table).inc();
    throw error;
  }
}

// Usage
const user = await trackQuery('SELECT', 'users', () => 
  db.query('SELECT * FROM users WHERE id = ?', [42])
);
```

## 3. Error Tracking

### Error Logging Best Practices

```javascript
// ❌ Bad: No context
console.error('Error');

// ❌ Bad: Only message
logger.error(error.message);

// ✅ Good: Full context
logger.error('Payment processing failed', {
  errorMessage: error.message,
  errorStack: error.stack,
  errorCode: error.code,
  userId: req.user.id,
  orderId: order.id,
  amount: order.amount,
  paymentMethod: order.paymentMethod,
  timestamp: new Date().toISOString(),
  correlationId: req.correlationId,
  endpoint: req.path,
  requestBody: sanitize(req.body) // Remove sensitive data
});
```

### Centralized Error Handler

```javascript
// Error handling middleware (must be last)
app.use((err, req, res, next) => {
  // Log error with full context
  logger.error('Request error', {
    error: {
      message: err.message,
      stack: err.stack,
      code: err.code || 'UNKNOWN'
    },
    request: {
      method: req.method,
      url: req.url,
      headers: sanitizeHeaders(req.headers),
      body: sanitize(req.body),
      query: req.query,
      params: req.params
    },
    user: {
      id: req.user?.id || null,
      ip: req.ip
    },
    correlationId: req.correlationId,
    timestamp: new Date().toISOString()
  });
  
  // Track error metric
  errorCounter.labels(err.code || 'UNKNOWN', req.path).inc();
  
  // Send to error tracking service (Sentry, Rollbar, etc.)
  Sentry.captureException(err, {
    extra: {
      correlationId: req.correlationId,
      userId: req.user?.id
    }
  });
  
  // Send error response
  res.status(err.statusCode || 500).json({
    error: {
      message: err.message,
      code: err.code || 'INTERNAL_ERROR',
      correlationId: req.correlationId
    }
  });
});
```

## 6. Potential Security Leaks During Logging

### Common Security Vulnerabilities

#### 1. **Logging Request Bodies (Credentials)**

```javascript
// ❌ DANGEROUS: Logs password
app.post('/login', (req, res) => {
  logger.info('Login attempt', {
    body: req.body // { email: 'user@example.com', password: 'secret123' }
  });
});

// ✅ SAFE: Sanitize before logging
function sanitize(obj) {
  const sanitized = { ...obj };
  const sensitiveFields = ['password', 'token', 'apiKey', 'secret', 'creditCard', 'ssn'];
  
  for (const field of sensitiveFields) {
    if (sanitized[field]) {
      sanitized[field] = '[REDACTED]';
    }
  }
  
  return sanitized;
}

app.post('/login', (req, res) => {
  logger.info('Login attempt', {
    body: sanitize(req.body) // { email: 'user@example.com', password: '[REDACTED]' }
  });
});
```

#### 2. **Logging Headers (Authorization)**

```javascript
// ❌ DANGEROUS: Logs auth token
logger.info('Request', {
  headers: req.headers // { authorization: 'Bearer eyJhbGc...' }
});

// ✅ SAFE: Redact sensitive headers
function sanitizeHeaders(headers) {
  const sanitized = { ...headers };
  const sensitiveHeaders = [
    'authorization',
    'x-api-key',
    'cookie',
    'x-auth-token',
    'x-csrf-token'
  ];
  
  for (const header of sensitiveHeaders) {
    if (sanitized[header]) {
      sanitized[header] = '[REDACTED]';
    }
  }
  
  return sanitized;
}

logger.info('Request', {
  headers: sanitizeHeaders(req.headers)
});
```

#### 3. **Logging Query Parameters (Sensitive Data)**

```javascript
// ❌ DANGEROUS: Password in URL
// GET /reset-password?token=secret-reset-token-123
logger.info('Request', {
  url: req.url // Logs token!
});

// ✅ SAFE: Sanitize URLs
function sanitizeUrl(url) {
  const parsed = new URL(url, 'http://dummy');
  const sensitiveParams = ['token', 'password', 'apiKey', 'secret'];
  
  for (const param of sensitiveParams) {
    if (parsed.searchParams.has(param)) {
      parsed.searchParams.set(param, '[REDACTED]');
    }
  }
  
  return parsed.pathname + parsed.search;
}

logger.info('Request', {
  url: sanitizeUrl(req.url) // /reset-password?token=[REDACTED]
});
```

#### 4. **Error Stack Traces (Database Credentials)**

```javascript
// ❌ DANGEROUS: Stack trace contains connection string
try {
  await db.connect('postgresql://admin:secret@db.example.com/mydb');
} catch (error) {
  logger.error('Database error', {
    stack: error.stack // Contains password!
  });
}

// ✅ SAFE: Sanitize stack traces
function sanitizeStack(stack) {
  return stack
    .replace(/\/\/([^:]+):([^@]+)@/g, '//[USER]:[REDACTED]@') // DB credentials
    .replace(/password[=:]\s*['"]?([^'"\s]+)['"]?/gi, 'password=[REDACTED]') // password=secret
    .replace(/api[_-]?key[=:]\s*['"]?([^'"\s]+)['"]?/gi, 'api_key=[REDACTED]'); // api_key=123
}

try {
  await db.connect('postgresql://admin:secret@db.example.com/mydb');
} catch (error) {
  logger.error('Database error', {
    message: error.message,
    stack: sanitizeStack(error.stack)
  });
}
```

### Comprehensive Sanitization Middleware

```javascript
const crypto = require('crypto');

class LogSanitizer {
  constructor() {
    this.sensitiveFields = new Set([
      'password',
      'token',
      'apiKey',
      'api_key',
      'secret',
      'creditCard',
      'credit_card',
      'cardNumber',
      'card_number',
      'cvv',
      'ssn',
      'social_security',
      'passport',
      'driverLicense',
      'authorization',
      'cookie',
      'session'
    ]);
  }
  
  sanitize(obj, options = {}) {
    if (typeof obj !== 'object' || obj === null) {
      return obj;
    }
    
    const sanitized = Array.isArray(obj) ? [] : {};
    
    for (const [key, value] of Object.entries(obj)) {
      const lowerKey = key.toLowerCase();
      
      // Check if field is sensitive
      if (this.isSensitive(lowerKey)) {
        if (options.hash) {
          sanitized[key] = this.hash(String(value));
        } else {
          sanitized[key] = '[REDACTED]';
        }
      }
      // Recursively sanitize nested objects
      else if (typeof value === 'object' && value !== null) {
        sanitized[key] = this.sanitize(value, options);
      }
      // Keep non-sensitive values
      else {
        sanitized[key] = value;
      }
    }
    
    return sanitized;
  }
  
  isSensitive(fieldName) {
    return this.sensitiveFields.has(fieldName) ||
           fieldName.includes('password') ||
           fieldName.includes('secret') ||
           fieldName.includes('token');
  }
  
  hash(value) {
    return crypto
      .createHash('sha256')
      .update(String(value))
      .digest('hex')
      .substring(0, 16);
  }
  
  sanitizeUrl(url) {
    try {
      const parsed = new URL(url, 'http://dummy');
      
      for (const param of parsed.searchParams.keys()) {
        if (this.isSensitive(param)) {
          parsed.searchParams.set(param, '[REDACTED]');
        }
      }
      
      return parsed.pathname + parsed.search;
    } catch {
      return url;
    }
  }
  
  sanitizeHeaders(headers) {
    const sanitized = {};
    
    for (const [key, value] of Object.entries(headers)) {
      if (this.isSensitive(key.toLowerCase())) {
        sanitized[key] = '[REDACTED]';
      } else {
        sanitized[key] = value;
      }
    }
    
    return sanitized;
  }
}

// Usage
const sanitizer = new LogSanitizer();

app.use((req, res, next) => {
  logger.info('Request', {
    method: req.method,
    url: sanitizer.sanitizeUrl(req.url),
    headers: sanitizer.sanitizeHeaders(req.headers),
    body: sanitizer.sanitize(req.body),
    query: sanitizer.sanitize(req.query)
  });
  
  next();
});
```

## Best Practices

### 1. **Use Structured Logging (JSON)**

```javascript
// ✅ Good: Easy to parse and search
logger.info('User login', {
  userId: 123,
  email: 'user@example.com',
  ip: '203.0.113.42',
  timestamp: new Date().toISOString()
});

// ❌ Bad: Hard to parse
console.log('User 123 (user@example.com) logged in from 203.0.113.42 at 2026-03-18T10:30:45Z');
```

### 2. **Set Appropriate Log Levels**

```javascript
// Production: Only important logs
if (process.env.NODE_ENV === 'production') {
  logger.level = 'info'; // info, warn, error
} else {
  logger.level = 'debug'; // Everything
}
```

### 3. **Include Context in Every Log**

```javascript
// ✅ Good: Full context
logger.error('Payment failed', {
  error: err.message,
  userId: user.id,
  orderId: order.id,
  amount: order.amount,
  correlationId: req.correlationId,
  timestamp: new Date().toISOString()
});

// ❌ Bad: No context
logger.error('Payment failed');
```

### 4. **Sanitize All Logs**

```javascript
// Always sanitize before logging
logger.info('Request', sanitizer.sanitize({
  ...req.body,
  headers: req.headers
}));
```

### 5. **Use Correlation IDs**

```javascript
// Track requests across services
logger.info('Processing order', {
  correlationId: req.correlationId, // Same ID across all logs
  orderId: order.id
});
```

### 6. **Log Rotation**

```javascript

require('winston-daily-rotate-file');

const transport = new winston.transports.DailyRotateFile({
```

### 7. **Centralized Logging**


### 8. **Monitor Log Volume**

```javascript

  name: 'logs_total',
```


## Complete Example: Production-Ready Logging

```javascript
const express = require('express');
const winston = require('winston');
const prometheus = require('prom-client');
const Sentry = require('@sentry/node');
const { v4: uuidv4 } = require('uuid');

const app = express();

// Initialize Sentry
Sentry.init({
  dsn: process.env.SENTRY_DSN,
  environment: process.env.NODE_ENV
});

// Configure logger
const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })
  ]
});

if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console({
    format: winston.format.simple()
  }));
}

// Prometheus metrics
const httpRequestDuration = new prometheus.Histogram({
  name: 'http_request_duration_seconds',
  help: 'Duration of HTTP requests in seconds',
  labelNames: ['method', 'route', 'status_code'],
  buckets: [0.1, 0.3, 0.5, 0.7, 1, 3, 5]
});

const httpRequestsTotal = new prometheus.Counter({
  name: 'http_requests_total',
  help: 'Total HTTP requests',
  labelNames: ['method', 'route', 'status_code']
});

// Sanitizer
const sanitizer = new LogSanitizer();

// Middleware: Correlation ID
app.use((req, res, next) => {
  req.correlationId = req.headers['x-correlation-id'] || uuidv4();
  res.setHeader('X-Correlation-ID', req.correlationId);
  next();
});

// Middleware: Request logging
app.use((req, res, next) => {
  const startTime = Date.now();
  
  // Log request
  logger.info('http_request', {
    correlationId: req.correlationId,
    method: req.method,
    url: sanitizer.sanitizeUrl(req.url),
    ip: req.ip,
    userAgent: req.get('user-agent'),
    userId: req.user?.id || null
  });
  
  // Log response
  res.on('finish', () => {
    const duration = (Date.now() - startTime) / 1000;
    
    logger.info('http_response', {
      correlationId: req.correlationId,
      method: req.method,
      url: req.url,
      statusCode: res.statusCode,
      duration
    });
    
    // Metrics
    httpRequestDuration
      .labels(req.method, req.route?.path || req.path, res.statusCode)
      .observe(duration);
      
    httpRequestsTotal
      .labels(req.method, req.route?.path || req.path, res.statusCode)
      .inc();
  });
  
  next();
});

// Routes
app.post('/api/users', async (req, res, next) => {
  try {
    logger.info('Creating user', {
      correlationId: req.correlationId,
      body: sanitizer.sanitize(req.body)
    });
    
    const user = await createUser(req.body);
    
    logger.info('User created', {
      correlationId: req.correlationId,
      userId: user.id
    });
    
    res.status(201).json(user);
    
  } catch (error) {
    next(error);
  }
});

// Metrics endpoint
app.get('/metrics', async (req, res) => {
  res.set('Content-Type', prometheus.register.contentType);
  res.end(await prometheus.register.metrics());
});

// Error handler
app.use((err, req, res, next) => {
  logger.error('request_error', {
    correlationId: req.correlationId,
    error: {
      message: err.message,
      stack: err.stack,
      code: err.code
    },
    request: {
      method: req.method,
      url: req.url,
      body: sanitizer.sanitize(req.body)
    },
    user: {
      id: req.user?.id || null
    }
  });
  
  Sentry.captureException(err, {
    extra: { correlationId: req.correlationId }
  });
  
  res.status(err.statusCode || 500).json({
    error: {
      message: err.message,
      correlationId: req.correlationId
    }
  });
});

app.listen(3000, () => {
  logger.info('Server started', { port: 3000 });
});
```

## Summary

### Key Takeaways

1. **Structured Logging (JSON)**
   - Easy to search and analyze
   - `{ "userId": 123, "action": "login" }`

2. **Correlation IDs**
   - Track requests across services
   - Essential for debugging

3. **Sanitize Everything**
   - Never log passwords, tokens, credit cards
   - Use `[REDACTED]` or hash sensitive data

4. **Four Golden Signals**
   - Latency (p50, p95, p99)
   - Traffic (RPS)
   - Errors (error rate %)
   - Saturation (CPU, memory)

5. **Centralized Logging**
   - ELK Stack, CloudWatch, Datadog
   - Don't rely on server logs

6. **Security First**
   - Sanitize PII
   - Restrict log file access
   - Comply with GDPR/HIPAA

### What to Log
✅ Request metadata (method, URL, status, duration)
✅ Business events (order created, user registered)
✅ Errors with context
✅ Performance metrics
✅ Security events (failed logins, rate limits)

### What NOT to Log
❌ Passwords
❌ API keys / tokens
❌ Credit card numbers
❌ SSN / personal IDs
❌ Health information
❌ Full URLs with sensitive params

### Security Checklist
- [ ] Sanitize request bodies
- [ ] Redact authorization headers
- [ ] Sanitize URLs with tokens
- [ ] Clean stack traces
- [ ] Restrict log file permissions (600)
- [ ] Don't expose logs via HTTP
- [ ] Use HTTPS for log shipping
- [ ] Encrypt logs at rest
- [ ] Set log retention policies
- [ ] Monitor log access

**Remember: Logs are as sensitive as your database. Treat them with the same security standards!**
